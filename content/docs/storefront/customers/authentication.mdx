# Customer Authentication

Storefront provides a complete authentication system for your customers. When a customer successfully authenticates, the API returns a **JSON Web Token (JWT)**. Your frontend app stores this token and sends it in the `Authorization` header for all authenticated API requests.

## Authentication Methods

### SMS Verification (OTP)

The most common method for on-demand delivery apps. No password required — a one-time passcode is sent to the customer's phone.

Flow:
1. Customer enters their phone number
2. Storefront sends an SMS OTP
3. Customer enters the OTP in the app
4. If valid, the customer is authenticated (account created automatically if new)

### Email & Password

Traditional registration with email address and password.

### Social Login (OAuth)

Storefront supports sign-in via:
- **Apple Sign-In** — required by Apple for iOS apps that offer other social logins
- **Google**
- **Facebook**

To enable social login, configure the OAuth credentials in **Storefront → Settings → Authentication** with the respective client IDs and secrets.

## Login API

### SMS OTP Login

```http
POST /storefront/v1/customers/login-with-sms
Storefront-Key: store_your_key

{
  "phone": "+15551234567"
}
```

After the OTP is sent, verify it:

```http
POST /storefront/v1/customers/verify-sms
Storefront-Key: store_your_key

{
  "phone": "+15551234567",
  "code": "123456"
}
```

### Email Login

```http
POST /storefront/v1/customers/login
Storefront-Key: store_your_key

{
  "email": "customer@example.com",
  "password": "securepassword"
}
```

### Response

All login endpoints return the same structure on success:

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5...",
  "customer": {
    "id": "customer_abc123",
    "name": "Jane Doe",
    "email": "customer@example.com",
    "phone": "+15551234567"
  }
}
```

Store the token securely (e.g., in device keychain or `AsyncStorage`). Include it in subsequent requests:

```http
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5...
```

## Registration

```http
POST /storefront/v1/customers
Storefront-Key: store_your_key

{
  "name": "Jane Doe",
  "email": "customer@example.com",
  "phone": "+15551234567",
  "password": "securepassword"
}
```

## Token Expiry & Sessions

JWTs have an expiration time. Handle `401 Unauthorized` responses by prompting re-authentication. Customers can also explicitly log out, which invalidates the token server-side.

<Callout type="info">
If you are using the open-source Storefront App, authentication is already fully implemented with SMS OTP, Apple Sign-In, and Google — no additional setup needed beyond configuring the OAuth credentials in the console.
</Callout>
