# Customer Authentication

Storefront provides a robust authentication system for your customers, supporting both traditional password-based login and modern passwordless methods.

When a customer successfully authenticates, the [Storefront API](/docs/api-reference/storefront) returns a secure JSON Web Token (JWT). Your frontend application must store this token and include it in the `Authorization` header of all subsequent API requests made on behalf of that customer.

## Authentication Methods

Storefront supports several authentication strategies out of the box:

### 1. Email & Password
The traditional method. Customers register with an email address and a secure password.

### 2. SMS Verification (OTP)
A popular, frictionless method for mobile apps.
1. The customer enters their phone number.
2. Storefront generates a One-Time Password (OTP) and sends it via SMS (using your configured [Notifications](/docs/storefront/settings/notifications) provider, such as Twilio).
3. The customer enters the OTP into your app.
4. If the OTP is valid, Storefront authenticates the user (and creates a new account if one doesn't exist).

### 3. Social Login (OAuth)
Storefront supports Single Sign-On (SSO) via major providers:
- **Apple Sign-In** (Required by Apple for iOS apps that offer other social logins)
- **Google**
- **Facebook**

To use social login, you must configure the respective OAuth credentials in the Fleetbase Console under **Storefront** > **Settings** > **Authentication**.

## Implementing Authentication in Your App

If you are using the open-source [Storefront App](/docs/apps/storefront-app), all of these authentication methods are already implemented and ready to use.

If you are building a custom frontend, you will interact with the `/v1/customers/login` and `/v1/customers/register` endpoints of the Storefront API.

### Example: Email Login

```javascript
// Example using fetch() in a JavaScript frontend
const response = await fetch('https://api.fleetbase.io/v1/storefront/customers/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Storefront-Key': 'YOUR_PUBLIC_STOREFRONT_KEY'
  },
  body: JSON.stringify({
    email: 'customer@example.com',
    password: 'securepassword123'
  })
});

const data = await response.json();

if (response.ok) {
  // Store the token securely (e.g., in AsyncStorage or HttpOnly cookie)
  const token = data.token;
  console.log('Logged in successfully!');
} else {
  console.error('Login failed:', data.error);
}
```

## Managing Sessions

The JWT returned by Storefront has an expiration time. If the token expires, the customer will need to log in again.

Your frontend application should handle `401 Unauthorized` responses from the API gracefully, prompting the user to re-authenticate or automatically refreshing the token if you have implemented a refresh token strategy.

Customers can also explicitly log out, which invalidates their current token on the server.
