# Products

A **Product** is the core unit of your catalog — any item or service a customer can purchase. Products belong to a specific store and are organized into categories.

Navigate to **Storefront → Products** to manage your product list.

![Product list — name, price, category, status, and primary image](/images/placeholder.png)

## Product Attributes

| Field | Description |
| :--- | :--- |
| **Name** | The customer-facing product name |
| **Description** | Detailed description of the item |
| **Price** | Base price in the store's currency |
| **Sale Price** | Reduced price; applied when `is_on_sale` is enabled |
| **SKU** | Optional internal stock keeping unit |
| **Category** | The category this product belongs to |
| **Status** | `available` (visible) or `draft` (hidden) |
| **Is Recommended** | Flag to surface this product as a recommendation in your app |
| **Is Service** | Mark as a service product (non-physical) |
| **Is Bookable** | Enable appointment/booking-style ordering |
| **Can Pickup** | Allow customers to select in-store pickup for this product |
| **Tags** | Freeform tags for filtering and search |

## Creating a Product

<Steps>
  <Step>
    Navigate to **Storefront → Products**.
  </Step>
  <Step>
    Click **New Product**.
  </Step>
  <Step>
    Enter the **Name**, **Description**, and **Price**.
  </Step>
  <Step>
    Select a **Category** from the dropdown (create one first if needed).
  </Step>
  <Step>
    Upload a **Primary Image** and any additional photos.
  </Step>
  <Step>
    Set **Status** to `available` when the product is ready for customers to see.
  </Step>
  <Step>
    Click **Save**.
  </Step>
</Steps>

## Product Customization

### Variants

Variants define required selections the customer must make before adding a product to the cart (e.g., selecting a size). See [Variants & Addons](/docs/storefront/catalog/variants-and-addons).

### Addons

Addons are optional extras grouped into addon categories. See [Variants & Addons](/docs/storefront/catalog/variants-and-addons).

### Product Hours

If a product is only available at certain times (e.g., a breakfast item available 7am–11am only):

1. Open the product and click the **Hours** tab
2. Click **Add Hours**
3. Select the day and set the start/end time
4. Save

When a customer tries to add an unavailable product to the cart outside its hours, the API will reject the request.

## How Products Flow to Fleet-Ops

When a customer completes checkout, each cart item is converted into a Fleet-Ops **Entity** in the order payload:

- The entity carries the product name, SKU, quantity, price, and selected variants/addons
- Dispatchers and drivers see exactly what needs to be picked up
- Entities can have their own tracking numbers and QR codes

This conversion happens automatically — no configuration required.

## Importing Products

To bulk-upload a product catalog from a spreadsheet, click **Import** in the products list. The CSV format requires at minimum: `name`, `price`, `currency`, and `category`.
