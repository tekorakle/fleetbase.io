# Gateway Settings

**Gateways** are the payment processors your store uses to accept payments at checkout. Configure gateways in **Storefront → Settings → Gateways**.

Gateways can be configured at the **store level** (applies to one store) or the **network level** (applies to all stores in the network).

## Adding a Gateway

<Steps>
  <Step>
    Navigate to **Storefront → Settings → Gateways**.
  </Step>
  <Step>
    Click **Add Gateway**.
  </Step>
  <Step>
    Select the gateway type: **Stripe**, **QPay**, or **Cash**.
  </Step>
  <Step>
    Enter the required credentials (see per-gateway config below).
  </Step>
  <Step>
    Toggle **Sandbox Mode** on while testing.
  </Step>
  <Step>
    Click **Save**.
  </Step>
</Steps>

## Stripe Configuration

| Field | Description |
| :--- | :--- |
| **Publishable Key** | Stripe public key (starts with `pk_test_` or `pk_live_`) |
| **Secret Key** | Stripe secret key (starts with `sk_test_` or `sk_live_`) |
| **Sandbox Mode** | On = test mode; Off = live mode |
| **Return URL** | URL Stripe redirects to after off-session payments |
| **Webhook URL** | Auto-generated — paste this into Stripe Dashboard → Webhooks |

Register the webhook URL in your Stripe Dashboard so Stripe can notify Fleetbase of payment success/failure.

## QPay Configuration

| Field | Description |
| :--- | :--- |
| **Username** | QPay merchant username |
| **Password** | QPay merchant password |
| **Invoice Code** | QPay invoice type code from your merchant agreement |
| **Sandbox Mode** | On = test environment |

## Cash on Delivery

No credentials required. Configure a display name (e.g., "Pay on Delivery") and save. COD orders are created without payment capture — the driver collects cash and confirms receipt in the Navigator app.

## Removing a Gateway

Open the gateway from the list and click **Delete**. Existing orders tied to this gateway are unaffected.
