# Cart

The **Cart** is a persistent shopping session. Cart state is stored in the database, so customers can abandon and return without losing their selections. Carts expire after **7 days** of inactivity.

## Cart Sessions

Carts are tied to one of:
- An **authenticated customer** — retrieved automatically when the customer logs in
- A **guest session** — a unique identifier your app generates and stores locally (e.g., in `AsyncStorage`). Pass this as `unique_identifier` when making cart API requests.

## Cart Item Structure

Each item in the cart records:
- `product_id` — the product
- `store_id` and `store_location_id` — which store and location this item is from
- `quantity` — how many units
- `price` — unit price at time of adding
- `subtotal` — unit price × quantity + addon costs
- `variants` — array of selected variant option names
- `addons` — array of selected addon names and prices
- `scheduled_at` — optional future scheduling timestamp

## Adding Items

```http
POST /storefront/v1/carts/{cart_id}/{product_id}
Storefront-Key: store_your_key

{
  "quantity": 2,
  "variants": [{ "id": "variant_option_id" }],
  "addons": [{ "id": "addon_id" }, { "id": "addon_id_2" }]
}
```

Storefront validates that:
- The product is active (`status = available`)
- The store is currently open (within operating hours)
- The product is within its own availability hours (if set)

## Updating & Removing Items

```http
PUT /storefront/v1/carts/{cart_id}
```

Pass updated `quantity`, `variants`, or `addons` for an existing cart item. To remove an item, set `quantity` to `0` or call DELETE on the specific cart item.

## Cart Calculations

Cart totals are recalculated after every modification:

```
Item Subtotal = (Base Price + Variant Additional Costs + Addon Prices) × Quantity
Cart Subtotal = Sum of all item subtotals
```

<Callout type="info">
The delivery fee is **not** included in the cart. It is calculated during [Checkout](/docs/storefront/orders/checkout) once the customer provides a delivery address and Storefront requests a Fleet-Ops Service Quote.
</Callout>

## Multi-Store Cart Behavior

By default, a cart holds items from **one store only**. If a customer has items from Store A and tries to add an item from Store B, the API returns an error. Your frontend should handle this by prompting:

> "Your cart has items from another store. Clear your cart to start a new order?"

This restriction exists because each store has its own location, operating hours, and dispatch requirements.

## Applying Discount Codes

Pass a `discount_code` on the cart to apply a promotion. If valid, the discount amount is reflected in the cart's total. See [Promotions](/docs/storefront/promotions/overview).
