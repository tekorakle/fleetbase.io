# Checkout

**Checkout** is the process of converting a cart into a paid order. It captures the customer's delivery address, calculates the delivery fee, processes payment, and creates a Fleet-Ops order — all in one atomic flow.

## Checkout Flow

### Step 1 — Delivery Address

The customer provides their delivery address (or selects a saved address). This is required before the delivery fee can be calculated.

### Step 2 — Service Quote

Storefront calls Fleet-Ops to request a **Service Quote** — the calculated delivery fee based on:
- Distance between the store location and the customer's delivery address
- The applicable Service Rate configured in Fleet-Ops

The quote is returned to the frontend and displayed to the customer as the `delivery_fee`.

### Step 3 — Apply Promotions

If the customer has a discount code, it is validated now:
- Is the promotion currently active?
- Has the usage limit been reached?
- Does the cart meet the minimum spend requirement?

If valid, the discount is applied to the subtotal.

### Step 4 — Payment

The customer selects a payment method. Depending on the configured gateways:
- **Stripe** — frontend collects card details via Stripe SDK, sends directly to Stripe
- **QPay** — customer is redirected to QPay payment page
- **Cash on Delivery** — no payment now; driver collects cash on delivery

### Step 5 — Order Creation

On payment success, your frontend calls the checkout capture endpoint. Storefront performs final validation:
- Is the store still online?
- Are all cart products still active?
- Was payment confirmed?

If everything is valid:
1. The cart is finalized
2. A **Checkout** record is created (snapshot of cart state + payment)
3. A **Fleet-Ops Order** is created with pickup = store location, dropoff = customer address, payload = cart items
4. The order ID is returned to the frontend for the confirmation screen

## Pickup Orders

If the customer selects **in-store pickup** (supported stores only):
- No delivery fee is calculated
- No Fleet-Ops driver dispatch occurs
- The order status transitions based on store preparation and customer pickup confirmation

## Checkout API

```http
POST /storefront/v1/checkouts/capture
Storefront-Key: store_your_key
Authorization: Bearer customer_token

{
  "cart_id": "cart_abc123",
  "delivery_address": {
    "street": "123 Main St",
    "city": "San Francisco",
    "country": "US",
    "coordinates": [-122.4194, 37.7749]
  },
  "gateway_id": "gateway_stripe_id",
  "is_pickup": false
}
```

On success, returns the created Fleet-Ops order object with tracking information.

## Required Minimum Amount

Stores can optionally configure a **Minimum Checkout Amount** in **Storefront → [Store] → Details → Options**. If the cart subtotal is below this threshold, checkout is blocked.
