# Payment Gateways

To accept payments, configure one or more **Payment Gateways** in your store or network settings. Storefront supports Stripe, QPay, and Cash on Delivery out of the box.

Navigate to **Storefront → Settings → Gateways** to configure gateways.

## Supported Gateways

| Gateway | Type | Notes |
| :--- | :--- | :--- |
| **Stripe** | Card / digital payments | Global; supports test mode |
| **QPay** | Invoice-based payments | Mongolia and regional |
| **Cash on Delivery** | No gateway | Driver collects cash on delivery |

## Configuring Stripe

<Steps>
  <Step>
    Create a [Stripe account](https://stripe.com) if you don't have one.
  </Step>
  <Step>
    In the Stripe Dashboard, copy your **Publishable Key** and **Secret Key**. Use test keys during development.
  </Step>
  <Step>
    Navigate to **Storefront → Settings → Gateways**.
  </Step>
  <Step>
    Click **Add Gateway** and select **Stripe**.
  </Step>
  <Step>
    Enter the **Publishable Key** and **Secret Key**.
  </Step>
  <Step>
    Enable **Sandbox Mode** while testing.
  </Step>
  <Step>
    Click **Save**.
  </Step>
</Steps>

## Stripe Payment Flow

When a customer reaches checkout:

1. Frontend requests a payment setup from the Storefront API
2. Storefront creates a Stripe `PaymentIntent` using your secret key
3. The `client_secret` is returned to the frontend
4. Frontend uses the Stripe SDK to securely collect card details
5. Stripe processes the payment and fires a webhook to Storefront
6. On webhook confirmation, Storefront finalizes the checkout and creates the Fleet-Ops order

Raw card numbers never touch the Fleetbase API — PCI compliance is maintained.

## Configuring QPay

<Steps>
  <Step>
    Navigate to **Storefront → Settings → Gateways**.
  </Step>
  <Step>
    Click **Add Gateway** and select **QPay**.
  </Step>
  <Step>
    Enter your QPay credentials (username, password, invoice code from your QPay merchant account).
  </Step>
  <Step>
    Click **Save**.
  </Step>
</Steps>

## Configuring Cash on Delivery

<Steps>
  <Step>
    Navigate to **Storefront → Settings → Gateways**.
  </Step>
  <Step>
    Click **Add Gateway** and select **Cash**.
  </Step>
  <Step>
    Enter a display name (e.g., "Pay on Delivery").
  </Step>
  <Step>
    Click **Save**.
  </Step>
</Steps>

COD orders are created immediately without payment processing. Payment status is `pending` until the driver marks it collected in the Navigator app.

## Sandbox Mode

All gateways except Cash support a **Sandbox** (test) mode. Enable sandbox during development — no real charges are made. Switch to live credentials before going to production.

## Gateway per Store vs. per Network

Gateways can be configured at the **store level** or the **network level**:
- Store-level gateways are available only to that store's checkout
- Network-level gateways are available to all stores within the network

Customers browsing a network see the gateway options configured on the network.
