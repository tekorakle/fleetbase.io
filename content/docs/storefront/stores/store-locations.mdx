# Store Locations

A **Store Location** is the physical address of a store. When a customer places an order, the location address becomes the **pickup address** in the Fleet-Ops order — the point where the driver collects the items for delivery.

A single store can have multiple locations (e.g., a franchise with five branches). Storefront automatically routes incoming orders to the nearest open location.

Navigate to **Storefront → [Store] → Locations** to manage locations.

![Store locations tab — list of locations with addresses, hours, and status](/images/placeholder.png)

## Adding a Location

<Steps>
  <Step>
    Open your store and click the **Locations** tab.
  </Step>
  <Step>
    Click **Add Location**.
  </Step>
  <Step>
    Enter a **Name** (e.g., "Downtown Branch", "Warehouse A").
  </Step>
  <Step>
    Enter the **Address**. As you type, Fleetbase geocodes the address and drops a pin on the map.
  </Step>
  <Step>
    If geocoding fails, drag the map pin to the correct position or enter **Latitude/Longitude** manually.
  </Step>
  <Step>
    Click **Save**.
  </Step>
</Steps>

After adding a location, set its **Operating Hours** — the days and times when this location accepts orders. See [Operating Hours](/docs/storefront/stores/operating-hours).

## Multiple Locations and Order Routing

When a store has multiple locations, Storefront determines which location fulfills an order using a nearest-neighbor algorithm:

1. Customer enters their delivery address at checkout
2. Storefront calculates distance from the delivery address to every active location
3. The order is assigned to the nearest open location (based on operating hours)
4. If the nearest location is closed, the next nearest is checked

If all locations are closed or outside range, checkout is blocked and the customer is informed.

## Delivery Zones

By default, a store location can deliver to any address within the radius configured in Fleet-Ops routing settings.

To restrict deliveries to a specific geographic area, link the location to a Fleet-Ops **Service Area** or **Zone**:
- Open the location → **Zone** field → select a zone
- Orders outside that zone will be rejected at checkout

Zones are created in **Fleet-Ops → Resources → Service Areas**. See the [Fleet-Ops Service Areas](/docs/fleet-ops/resources/places/overview) documentation.

## Food Trucks

For mobile stores, the location coordinates can be updated dynamically — the store's position shifts as the food truck moves. See [Food Trucks](/docs/storefront/stores/food-trucks) for the full mobile store setup guide.
