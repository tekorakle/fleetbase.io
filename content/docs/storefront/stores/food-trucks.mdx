# Food Trucks

Storefront supports **mobile stores** — food trucks, pop-up shops, and any merchant whose location changes day to day. A food truck in Storefront is a store with a location linked directly to a **Fleet-Ops Vehicle**, so its position updates in real-time as the vehicle moves.

Navigate to **Storefront → Food Trucks** to manage mobile stores.

![Food trucks list — store name, vehicle, current location, and status](/images/placeholder.png)

## How Food Trucks Work

A food truck is a **FoodTruck** record that links:
- A **Store** — the merchant and catalog
- A **Vehicle** — from Fleet-Ops Resources, providing the live GPS position
- An optional **Catalog** — the specific menu available from this truck (useful when one store runs multiple trucks with different menus)
- An optional **Service Area** or **Zone** — restricts where the truck accepts orders

When a customer places an order, the truck's current GPS position from the Fleet-Ops vehicle is used as the pickup address.

## Creating a Food Truck

<Steps>
  <Step>
    Ensure the vehicle that represents the food truck exists in **Fleet-Ops → Resources → Vehicles**.
  </Step>
  <Step>
    Navigate to **Storefront → Food Trucks**.
  </Step>
  <Step>
    Click **New Food Truck**.
  </Step>
  <Step>
    Select the **Store** this food truck belongs to.
  </Step>
  <Step>
    Select the **Vehicle** from Fleet-Ops that represents the physical truck.
  </Step>
  <Step>
    Optionally assign a **Catalog** — if this truck serves a specific subset menu.
  </Step>
  <Step>
    Optionally assign a **Service Area** or **Zone** to restrict the delivery area.
  </Step>
  <Step>
    Click **Save**.
  </Step>
</Steps>

## Catalogs for Food Trucks

A **Catalog** is a named collection of product categories. You can create multiple catalogs for the same store and assign different catalogs to different food trucks — useful when one brand operates trucks with different menus (e.g., a BBQ truck and a taco truck under the same store).

To create a catalog:
1. Navigate to **Storefront → Catalogs**
2. Click **New Catalog**
3. Select the categories and products to include
4. Set availability hours for the catalog if needed
5. Assign the catalog to a food truck

## Real-Time Location Updates

The food truck's position in the customer-facing app comes directly from the Fleet-Ops vehicle's GPS telemetry. As the vehicle moves, the truck's position updates automatically — no manual location entry required.

Customers browsing the network will see the food truck's current live position on the map.

## Operating Hours for Food Trucks

Set operating hours on the **store location** or **catalog** to define when the truck accepts orders at a given position. If the truck moves to a new neighborhood, update its zone assignment if you use zone-based delivery restrictions.
