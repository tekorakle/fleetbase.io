# Push Notifications

Push notifications let you re-engage customers, announce new products, and alert them about active promotions. Storefront's notification engine supports **Apple Push Notification service (APNs)** for iOS and **Firebase Cloud Messaging (FCM)** for Android.

## Prerequisites

Before sending push notifications you must configure a notification channel.

Navigate to **Storefront → Settings → Notifications** and add a channel. See [Notification Settings](/docs/storefront/settings/notifications) for setup.

Your frontend app must also:
1. Request notification permissions from the customer
2. Obtain the device token from APNs or FCM
3. Register the device token with the Storefront API so the system knows which devices belong to which customer

## Sending a Broadcast Notification

A **broadcast** sends to all customers who have opted in to notifications.

<Steps>
  <Step>
    Navigate to **Storefront → Promotions → Push Notifications**.
  </Step>
  <Step>
    Click **New Notification**.
  </Step>
  <Step>
    Enter the **Title** (bold headline) and **Body** (message text).
  </Step>
  <Step>
    Optionally attach an **Image** for rich media notifications.
  </Step>
  <Step>
    Optionally set a **Deep Link** — the route within your app that opens when the customer taps the notification (e.g., `myapp://category/pizza`).
  </Step>
  <Step>
    Click **Send**. Storefront queues the notification and dispatches it to all registered devices.
  </Step>
</Steps>

## Targeted Notifications

Instead of broadcasting to everyone, send to specific customers:

1. When creating a notification, use the **Recipients** selector
2. Search for and select specific customers
3. Only those customers' devices receive the notification

Use targeted notifications to:
- Re-engage customers who haven't ordered in 30+ days
- Send special offers to your top buyers
- Notify specific customers of a resolution to a service issue

## Automated Transactional Notifications

Storefront automatically sends order status notifications as the Fleet-Ops order progresses — no manual setup required:

- **Order accepted** — merchant accepted the order and is preparing
- **Driver dispatched** — driver heading to pickup
- **Order in transit** — driver collected items, en route
- **Order delivered** — delivery confirmed

These are triggered by Fleet-Ops order status changes and delivered via your configured notification channel.

<Callout type="info">
Transactional notifications fire automatically as long as a notification channel is configured and the customer's device token is registered. They do not require any manual action.
</Callout>
