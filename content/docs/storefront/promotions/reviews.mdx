# Reviews & Ratings

The **Reviews** system lets authenticated customers leave ratings and text feedback on both products and stores. Reviews build trust and help customers make informed purchasing decisions.

Navigate to **Storefront → Promotions → Reviews** to manage submitted reviews.

![Reviews list — reviewer name, subject, rating stars, content, submission date](/images/placeholder.png)

## What a Review Contains

| Field | Description |
| :--- | :--- |
| **Rating** | 1–5 star numeric score |
| **Content** | Optional text feedback |
| **Subject** | The product or store being reviewed |
| **Author** | The authenticated customer who submitted it |
| **Media** | Optional photo or video uploads attached to the review |
| **Votes** | Other customers can vote the review as helpful or unhelpful |

## Submitting Reviews (API)

Only authenticated customers can submit reviews. Call the reviews endpoint after an order is delivered:

```http
POST /storefront/v1/reviews
Authorization: Bearer customer_token
Storefront-Key: store_your_key

{
  "subject_type": "product",
  "subject_id": "product_abc123",
  "rating": 5,
  "content": "Arrived hot and fresh — exactly as described!"
}
```

For store reviews, use `"subject_type": "store"` and the store's public ID.

## Product Reviews vs Store Reviews

- **Product reviews** — tied to a specific item; shown on the product detail page in your app
- **Store reviews** — tied to the store as a whole; shown in marketplace listings and on the store's profile page

Both appear in the reviews management panel in the console.

## Ratings & Averages

Storefront automatically computes and caches the **average rating** and **review count** for every product and store. These are included in all product and store API responses:

```json
{
  "id": "product_abc123",
  "name": "Classic Cheeseburger",
  "rating": 4.7,
  "reviews_count": 132
}
```

Your frontend can display these immediately without fetching individual reviews.

## Moderation

All submitted reviews appear in **Storefront → Promotions → Reviews**. Moderation options:

- **Reject** — marks the review as rejected; it is hidden from the public API but preserved in the system
- **Delete** — permanently removes the review

To only show approved reviews by default, configure your frontend to filter by `rejected: false` when fetching reviews. Rejected reviews are excluded from rating calculations.

## Voting on Reviews

Customers can vote on existing reviews to surface the most helpful feedback:

```http
POST /storefront/v1/reviews/{review_id}/vote
Authorization: Bearer customer_token

{
  "type": "up"
}
```

Vote types: `up` (helpful) or `down` (not helpful). Vote counts are included in the review response.
