# Contributing to Fleetbase

Fleetbase is open-source and welcomes contributions from the community. Whether you're fixing a typo, reporting a bug, or building a new feature, every contribution matters.

## Ways to Contribute

| Contribution type | Where to start |
| :--- | :--- |
| **Bug report** | [Open an issue](https://github.com/fleetbase/fleetbase/issues/new) on the relevant repo |
| **Feature request** | [Start a discussion](https://github.com/fleetbase/fleetbase/discussions) before opening a PR |
| **Code fix or feature** | Fork → branch → PR (see below) |
| **Documentation** | Same PR flow — docs live in this repo |
| **Extension** | See [Building Extensions](/docs/extending-fleetbase) |
| **Translation** | Edit `translations/*.yaml` files in the console repo |

## Reporting Bugs

A good bug report makes it easy for maintainers to reproduce and fix the problem. Include:

- **Summary** — what you expected to happen and what actually happened
- **Steps to reproduce** — specific, numbered steps
- **Environment** — Fleetbase version, deployment method (Docker, cloud, local), OS
- **Logs or error messages** — paste relevant output from `docker compose logs`
- **Code sample** — if relevant, a minimal reproducer

Open the report on the GitHub repository for the affected component:

| Repository | What it covers | Issues |
| :--- | :--- | :--- |
| **fleetbase/fleetbase** | Core platform — API, Console, IAM, Developer Console | [Open issue →](https://github.com/fleetbase/fleetbase/issues/new) |
| **fleetbase/fleetops** | FleetOps extension — fleet management, dispatch, tracking | [Open issue →](https://github.com/fleetbase/fleetops/issues/new) |
| **fleetbase/storefront** | Storefront extension — e-commerce, catalog, orders | [Open issue →](https://github.com/fleetbase/storefront/issues/new) |
| **fleetbase/pallet** | Pallet extension — warehouse and inventory management | [Open issue →](https://github.com/fleetbase/pallet/issues/new) |
| **fleetbase/ledger** | Ledger extension — accounting and invoicing | [Open issue →](https://github.com/fleetbase/ledger/issues/new) |
| **fleetbase/navigator-app** | Navigator driver mobile app (React Native) | [Open issue →](https://github.com/fleetbase/navigator-app/issues/new) |
| **fleetbase/storefront-app** | Storefront customer mobile app (React Native) | [Open issue →](https://github.com/fleetbase/storefront-app/issues/new) |

## Submitting a Pull Request

<Steps>
  <Step>
    ### Fork and clone

    Fork the repository you want to contribute to on GitHub, then clone your fork:

    ```bash
    git clone https://github.com/YOUR_USERNAME/fleetbase.git
    cd fleetbase
    ```
  </Step>
  <Step>
    ### Create a branch

    Branch from `main` with a descriptive name:

    ```bash
    git checkout -b fix/order-status-not-updating
    # or
    git checkout -b feat/add-bulk-driver-import
    ```
  </Step>
  <Step>
    ### Make your changes

    Follow the coding conventions for the component you're changing:

    - **PHP/Laravel (core-api)** — PSR-12 coding standard, type hints, docblocks on public methods
    - **Ember.js (console, extensions)** — follow the existing component and service patterns, native classes, no jQuery
    - **React Native (mobile apps)** — functional components, hooks, no class components
  </Step>
  <Step>
    ### Run tests

    ```bash
    # PHP tests (from the api container)
    docker compose exec application php artisan test

    # Ember tests (from the console directory)
    cd console && ember test
    ```

    Ensure all existing tests pass before submitting.
  </Step>
  <Step>
    ### Open the pull request

    Push your branch and open a PR against the `main` branch of the upstream repository. In the PR description:

    - Reference the issue it closes (e.g. `Closes #1234`)
    - Describe what changed and why
    - Note any breaking changes
    - Include screenshots for UI changes
  </Step>
</Steps>

## Building Extensions

The most impactful way to contribute is building extensions that other operators can install. Extensions are full-stack packages — a Laravel backend package and an Ember.js frontend engine — registered through the Fleetbase extension system.

See [Building Extensions](/docs/extending-fleetbase) for the full developer guide, including how to scaffold, test, and publish an extension to the registry.
