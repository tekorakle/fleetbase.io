# Community & Support

Fleetbase is open-source and community-driven. Whether you need help getting started, want to report a bug, contribute code, or talk through a deployment with the core team — this is where to start.

## Real-Time Help

**[Discord Community](/docs/community/discord)** — the fastest way to get an answer. The `#help` channel is monitored by the core team and experienced community members. For installation issues, API questions, and extension development, Discord is the first stop.

**[GitHub Discussions](https://github.com/fleetbase/fleetbase/discussions)** — best for longer-form questions, feature proposals, and conversations that don't have a natural end. See [GitHub Discussions](/docs/community/github-discussions) for how to use it effectively.

## Bug Reports & Feature Requests

Open an issue on the relevant GitHub repository:

| Repository | What it covers |
| :--- | :--- |
| [fleetbase/fleetbase](https://github.com/fleetbase/fleetbase/issues) | Core platform, API, Console |
| [fleetbase/fleetops](https://github.com/fleetbase/fleetops/issues) | FleetOps extension |
| [fleetbase/storefront](https://github.com/fleetbase/storefront/issues) | Storefront extension |
| [fleetbase/navigator-app](https://github.com/fleetbase/navigator-app/issues) | Navigator driver app |
| [fleetbase/storefront-app](https://github.com/fleetbase/storefront-app/issues) | Storefront customer app |

Before opening an issue, check the [FAQ](/docs/community/faq) and search existing issues — your problem may already have a solution or a pending fix.

## Contributing

Fleetbase welcomes contributions of all kinds. The [Contributing Guide](/docs/contributing) has dedicated tracks for:

- **[Translations](/docs/contributing/translations)** — translate the Console and extensions into your language
- **[Code](/docs/contributing/code)** — submit bug fixes and features
- **[Documentation](/docs/contributing/documentation)** — improve these docs
- **[Extensions](/docs/contributing/extensions)** — build for the marketplace
- **[Reporting issues](/docs/contributing/reporting-issues)** — file effective bug reports and feature requests

## Stay Updated

- **[Changelog](/docs/community/changelog)** — release notes for each Fleetbase repository
- **Twitter/X** — follow [@fleetbase_io](https://twitter.com/fleetbase_io) for announcements and news
- **Discord `#announcements`** — major releases and platform updates posted directly

## Licensing & Commercial Use

Fleetbase is released under **AGPL-3.0**. Self-hosting for internal or commercial operations is free. If you need to embed Fleetbase into a proprietary product or require white-labeling without releasing your source code, a commercial license is available.

See [Licensing](/docs/community/licensing) for a plain-language breakdown of what AGPL-3.0 means for your use case.
