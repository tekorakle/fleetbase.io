# Backend Development

The backend of a Fleetbase extension is a standard **PHP Laravel Package**.

If you are familiar with Laravel development, building a Fleetbase extension backend will feel very natural. You have full access to Eloquent ORM, routing, middleware, and the service container.

## The Server Directory

When you [scaffold an extension](/docs/extending-fleetbase/scaffolding-your-extension), the CLI generates a `server/` directory containing the backend package.

```text
server/
├── src/
│   ├── Http/
│   │   ├── Controllers/
│   │   └── Requests/
│   ├── Models/
│   └── Providers/
│       └── MyExtensionServiceProvider.php
├── database/
│   └── migrations/
├── routes/
│   └── api.php
└── composer.json
```

## The Service Provider

The entry point for your backend package is the Service Provider (e.g., `MyExtensionServiceProvider.php`).

This is where you register your routes, load your migrations, and bind any services into the Laravel container.

```php
namespace MyOrg\MyExtension\Providers;

use Illuminate\Support\ServiceProvider;

class MyExtensionServiceProvider extends ServiceProvider
{
    public function boot()
    {
        // Load routes
        $this->loadRoutesFrom(__DIR__.'/../../routes/api.php');

        // Load migrations
        $this->loadMigrationsFrom(__DIR__.'/../../database/migrations');
    }

    public function register()
    {
        // Register bindings
    }
}
```

## Defining Routes

Your API routes should be defined in `server/routes/api.php`.

Fleetbase uses a specific routing structure to ensure that extension routes do not conflict with core routes or routes from other extensions.

You should wrap your routes in a group that applies the `fleetbase.api` middleware and prefixes the routes with your extension's namespace.

```php
use Illuminate\Support\Facades\Route;
use MyOrg\MyExtension\Http\Controllers\MyCustomController;

Route::group([
    'prefix' => 'my-extension/v1',
    'middleware' => ['fleetbase.api']
], function () {
    Route::get('custom-data', [MyCustomController::class, 'index']);
    Route::post('custom-data', [MyCustomController::class, 'store']);
});
```

This ensures that your routes are accessible at `https://api.your-fleetbase-instance.com/my-extension/v1/custom-data` and are protected by Fleetbase's standard API authentication.

## Database Migrations

If your extension requires custom database tables, you can define them in `server/database/migrations/`.

These migrations will be automatically run when a user installs your extension via the Fleetbase CLI or Registry.

```php
use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

class CreateCustomDataTable extends Migration
{
    public function up()
    {
        Schema::create('my_extension_custom_data', function (Blueprint $table) {
            $table->id();
            $table->string('name');
            $table->timestamps();
        });
    }

    public function down()
    {
        Schema::dropIfExists('my_extension_custom_data');
    }
}
```

## Models and Controllers

You can define your Eloquent models in `server/src/Models/` and your controllers in `server/src/Http/Controllers/`.

Because your extension runs within the Fleetbase environment, you can easily interact with core Fleetbase models (like `User`, `Organization`, or `Order`) by importing them into your controllers.

```php
namespace MyOrg\MyExtension\Http\Controllers;

use Illuminate\Http\Request;
use Fleetbase\Models\User;
use MyOrg\MyExtension\Models\CustomData;

class MyCustomController extends Controller
{
    public function index(Request $request)
    {
        // Access the currently authenticated user
        $user = $request->user();

        // Fetch custom data
        $data = CustomData::where('organization_id', $user->organization_id)->get();

        return response()->json($data);
    }
}
```

Once your backend API is built, you can move on to [Frontend Development](/docs/extending-fleetbase/frontend-development) to build the UI that interacts with it.
