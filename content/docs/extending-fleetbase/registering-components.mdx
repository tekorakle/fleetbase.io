# Registering Components & UI

Beyond adding [Dashboard Widgets](/docs/extending-fleetbase/dashboard-widgets) and [Menu Items](/docs/extending-fleetbase/menus-and-navigation), the **Universe Service** allows you to inject custom UI components directly into existing views within the Fleetbase Console or other extensions.

This is a powerful feature that allows you to extend the functionality of core modules without modifying their source code.

## The Component Registry

The Universe Service maintains a registry of components mapped to specific "injection points" across the platform.

When a view in Fleetbase (e.g., the Order Details panel in FleetOps) renders, it checks the Universe Service to see if any extensions have registered components for that specific injection point. If so, it renders them dynamically.

## Registering a Component

You register a component using the `registerComponent` method on the Universe Service.

You typically do this in an initializer or within the `init` method of your engine's application route.

```javascript
import Route from '@ember/routing/route';
import { inject as service } from '@ember/service';

export default class ApplicationRoute extends Route {
    @service universe;

    init() {
        super.init(...arguments);

        this.universe.registerComponent('fleetops.order.details.tabs', {
            component: 'my-extension-order-tab',
            title: 'My Extension Data',
            icon: 'box',
        });
    }
}
```

In this example, we are injecting a component named `my-extension-order-tab` into the `fleetops.order.details.tabs` injection point.

## Common Injection Points

While the list of injection points is constantly growing, here are some common ones you might use:

### FleetOps
- **`fleetops.order.details.tabs`**: Adds a new tab to the Order Details panel.
- **`fleetops.driver.details.tabs`**: Adds a new tab to the Driver Details panel.
- **`fleetops.vehicle.details.tabs`**: Adds a new tab to the Vehicle Details panel.

### Core Platform
- **`console.settings.tabs`**: Adds a new tab to the global Settings panel.
- **`console.admin.tabs`**: Adds a new tab to the global Admin panel.

## Building the Injected Component

When building the component that will be injected, you need to be aware of the context it will be rendered in.

For example, if you inject a component into `fleetops.order.details.tabs`, your component will automatically receive the `order` model as an argument (`@order`).

```handlebars
{{!-- my-extension-order-tab.hbs --}}
<div class="p-4">
    <h4>Custom Data for Order {{@order.public_id}}</h4>
    
    {{#if this.isLoading}}
        <Spinner />
    {{else}}
        <p>This order has {{this.customData.itemCount}} special items.</p>
    {{/if}}
</div>
```

```javascript
// my-extension-order-tab.js
import Component from '@glimmer/component';
import { tracked } from '@glimmer/tracking';
import { inject as service } from '@ember/service';
import { action } from '@ember/object';

export default class MyExtensionOrderTabComponent extends Component {
    @service fetch;
    @tracked customData;
    @tracked isLoading = true;

    constructor() {
        super(...arguments);
        this.loadData();
    }

    @action
    async loadData() {
        this.isLoading = true;
        try {
            // Fetch data specific to the order passed in via @order
            this.customData = await this.fetch.get(`my-extension/v1/order-data/${this.args.order.id}`);
        } catch (error) {
            console.error('Failed to load custom data', error);
        } finally {
            this.isLoading = false;
        }
    }
}
```

This allows your extension to seamlessly integrate its own data and functionality directly into the workflows of other modules.
