# Policies & Security

Fleetbase provides a robust set of security features and policies to ensure that your logistics data remains secure, compliant, and accessible only to authorized personnel.

## Two-Factor Authentication (2FA)

To protect user accounts from unauthorized access, Fleetbase supports Two-Factor Authentication (2FA). When enabled, users must provide a time-based one-time password (TOTP) generated by an authenticator app (such as Google Authenticator or Authy) in addition to their standard password.

### Enabling 2FA

Users can enable 2FA for their own accounts:

1. Click on your profile avatar in the top right corner of the Fleetbase Console.
2. Select **Account Settings**.
3. Navigate to the **Security** tab.
4. Click **Enable Two-Factor Authentication**.
5. Scan the provided QR code with your authenticator app.
6. Enter the 6-digit code generated by the app to verify and activate 2FA.

### Enforcing 2FA Organization-Wide

Administrators can enforce 2FA for all users within an organization. When enforced, any user who has not yet set up 2FA will be prompted to do so upon their next login.

1. Navigate to **Settings** > **Security** in the Fleetbase Console.
2. Toggle the **Require Two-Factor Authentication** setting.
3. Click **Save Changes**.

## API Security Policies

Fleetbase uses API keys to authenticate requests from external systems, mobile apps, and custom integrations. To ensure these keys are not misused, Fleetbase provides several security policies.

### Environment Scoping

Every API key is scoped to a specific environment:
- **Test Mode**: Keys operate on a separate, isolated dataset. Actions performed with a Test key do not affect production data.
- **Live Mode**: Keys operate on your production data.

Always use Test keys during development and integration testing.

### IP Whitelisting

To prevent unauthorized use of an API key, you can restrict its usage to specific IP addresses. If a request is made using the key from an IP address not on the whitelist, the request will be rejected.

1. Navigate to **Developers** > **API Keys**.
2. Select the API key you want to secure.
3. In the **IP Whitelist** section, enter the allowed IP addresses (e.g., the static IP of your ERP system).
4. Click **Save**.

## Data Encryption

Fleetbase employs industry-standard encryption to protect your data both in transit and at rest.

- **In Transit**: All communication between the Fleetbase Console, mobile apps, and the Core API is encrypted using TLS 1.2 or higher (HTTPS).
- **At Rest**: Sensitive data, such as passwords and API secrets, are hashed using bcrypt before being stored in the database.

## Audit Logging

To maintain accountability and support compliance requirements, Fleetbase logs all significant actions performed within the platform.

- **API Request Logs**: Every request made to the Fleetbase API is logged, including the endpoint, method, payload, response status, and the API key used. These logs are accessible in the **Developers** > **Request Logs** panel.
- **Activity Logs**: Changes to critical resources (such as creating an order, updating a driver's status, or modifying a user's role) are recorded in the Activity Log, providing a complete audit trail of who did what and when.
