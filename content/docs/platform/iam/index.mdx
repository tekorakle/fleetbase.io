# Identity & Access Management (IAM)

Fleetbase includes a comprehensive, dynamic Identity and Access Management (IAM) system built directly into the core platform. This system governs who can access your Fleetbase Console, what actions they can perform, and which data they can view across all installed extensions.

Because Fleetbase is a modular operating system, the IAM system is designed to be extensible. When you install a new extension (like FleetOps or Storefront), the IAM system automatically registers the new permissions and policies required by that extension, allowing you to manage access from a single, centralized interface.

## Core IAM Components

The Fleetbase IAM system is built around four primary concepts:

1. **Users**: Individual accounts that log into the Fleetbase Console or mobile apps.
2. **Roles**: Collections of permissions assigned to users (e.g., "Admin", "Dispatcher", "Driver").
3. **Permissions**: Granular rules that allow or deny specific actions (e.g., `create-order`, `delete-driver`).
4. **Organizations**: The top-level tenant that groups users, data, and settings together.

## Managing Access

As an administrator, you can manage access through the **Settings** panel in the Fleetbase Console.

- **[Users, Roles & Permissions](/docs/platform/iam/users-roles-permissions)**: Learn how to invite team members, create custom roles, and assign granular permissions.
- **[Organizations & Groups](/docs/platform/iam/organizations-and-groups)**: Understand how to manage multi-tenant operations and group users for easier administration.
- **[Policies & Security](/docs/platform/iam/policies-and-security)**: Configure advanced security settings, including Two-Factor Authentication (2FA) and API access policies.

## Developer Integration

For developers building custom extensions, the Fleetbase IAM system provides a robust set of tools for securing your API routes and UI components.

- **Backend**: Use Laravel middleware and policies to protect your extension's API endpoints based on the user's assigned roles and permissions.
- **Frontend**: Use the Ember.js `currentUser` service to conditionally render UI components (like buttons or tabs) based on the user's access level.

By leveraging the built-in IAM system, you can ensure that your custom extensions adhere to the same strict security standards as the core platform.
