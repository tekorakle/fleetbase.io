# Overview

Fleetbase is an open-source **Logistics and Supply Chain Operating System (LSOS)** — a modular, API-first platform that gives logistics operators and software developers production-ready infrastructure they can deploy on their own terms, extend without limits, and own completely.

At its core, Fleetbase provides the foundation every logistics business needs: multi-tenant organisation management, identity and access control, a real-time event bus, a REST API, and a developer console for managing integrations. On top of that foundation, you install the modules you need — FleetOps for fleet management, Storefront for e-commerce, Pallet for inventory, Ledger for accounting — or you build your own.

## Who Fleetbase Is For

Fleetbase is built for two types of people who are almost always working together:

**Operators** are logistics businesses — courier companies, on-demand delivery platforms, e-commerce brands, field service providers, and enterprise supply chains. They need a platform that can be shaped to their exact operations, rather than forcing their operations to fit the software.

**Developers** are the engineers who build and maintain logistics systems. They need a stable, well-architected API layer, an extensible UI framework, and the ability to add custom logic without fighting the platform.

Both groups get the same thing: full access to the source code, the ability to self-host on any infrastructure, and a platform designed to be extended rather than worked around.

## The Console

The Fleetbase Console is a web-based administration interface built with **Ember.js**. It is where operators manage their organisation day-to-day — dispatching orders, managing users, configuring integrations, and installing extensions.

The console is modular by design. Each extension registers its own routes, pages, and sidebar links into the console at runtime using the **Universe Service** — Fleetbase's central extensibility registry. When you install FleetOps, a fully featured fleet management interface appears in the console. When you install Storefront, a complete e-commerce management panel appears alongside it. No redeployment required.

<Callout type="info">The console runs as a single Ember application, with each extension mounted as an **Ember Engine** — an isolated sub-application that shares core services (auth, data, notifications) but keeps its own routes and components completely separate.</Callout>

## First-Party Extensions

Fleetbase ships with a set of officially maintained extensions that cover the most common logistics use cases.

<Accordions type="single" collapsible>
  <Accordion title="FleetOps — Fleet Management & TMS">
    FleetOps is the primary operations module. It provides a full Transportation Management System (TMS) including:

    - **Order management** with configurable workflows, custom order types, and proof of delivery (photos, signatures, QR codes)
    - **Real-time fleet tracking** with live maps, driver location streams, and activity feeds
    - **Dispatch tools** including a kanban-style order board for visual assignment management
    - **Driver and vehicle management** with assignment tracking, performance metrics, and communication
    - **Route optimisation** and service zone definition
    - **Vendor and partner integrations** for multi-carrier operations

    FleetOps is the right starting point for courier companies, on-demand delivery platforms, field service businesses, and any operation where drivers and vehicles are the core asset.
  </Accordion>
  <Accordion title="Storefront — Headless E-Commerce & Marketplace">
    Storefront is a headless e-commerce and marketplace engine. It powers the consumer-facing side of on-demand delivery platforms:

    - **Multi-vendor marketplace** support — create a Network of Stores (equivalent to how DoorDash or UberEats aggregates restaurants)
    - **Single-brand storefront** — deploy a D2C ordering app for a single merchant or franchise
    - **Product catalog management** with variants, add-ons, pricing, and categories
    - **Order and checkout flows** integrated directly with FleetOps for fulfilment
    - **Customer management** with authentication and order history
    - **Promotions, reviews, and push notifications** for customer engagement

    Storefront is headless — it exposes a REST API that you connect to your own consumer-facing mobile app or website.
  </Accordion>
  <Accordion title="Pallet — Inventory & Warehouse Management">
    Pallet is the inventory and warehouse management extension. It gives operators visibility into stock across all their locations:

    - Track inventory across stores, warehouses, dark stores, and food trucks
    - Manage stock levels, transfers, and adjustments
    - Integrate with Storefront for real-time product availability
    - Receive low-stock alerts and generate inventory reports

    Pallet is the right choice for any operation where inventory visibility is critical to fulfilment accuracy.
  </Accordion>
  <Accordion title="Ledger — Accounting & Invoicing">
    Ledger brings financial management into the Fleetbase console:

    - Generate invoices automatically from completed orders
    - Track revenue, settlements, and outstanding payments
    - Manage financial records across your organisation
    - Connect with external accounting systems

    Ledger is particularly useful for marketplace operators who need to settle payments between their platform and vendor partners.
  </Accordion>
  <Accordion title="Developer Console — API & Integration Tools">
    The Developer Console is a built-in extension that ships with every Fleetbase installation:

    - **API key management** — generate public and secret API keys for REST API access
    - **Webhook configuration** — subscribe to platform events and deliver them to your endpoints
    - **Socket event monitoring** — view and manage real-time WebSocket channels and event streams
    - **Request logs** — inspect API request history, response codes, and payloads for debugging
    - **System events** — monitor platform-level events across your organisation

    The Developer Console is the primary integration point for any application connecting to Fleetbase.
  </Accordion>
</Accordions>

## The Extension Marketplace

Beyond the first-party modules, Fleetbase has a community extension marketplace where developers can publish and discover extensions. Extensions can add new order types, integrate third-party services (maps, payments, SMS), add custom reporting, or build entirely new verticals on top of the Fleetbase platform.

You can browse and install extensions directly from within the Fleetbase Console under **Extensions → Browse**.

## Deployment Options

Fleetbase can be run anywhere you can run Docker.

<Tabs items={['Fleetbase Cloud', 'Self-Hosted', 'On-Premise']}>
  <Tab value="Fleetbase Cloud">
    The fastest way to get started. Fleetbase Cloud is a fully managed, multi-tenant deployment maintained by the Fleetbase team. You get:

    - Instant access — no infrastructure setup required
    - Automatic updates and security patches
    - Managed backups and uptime monitoring
    - All first-party extensions available

    [Start with Cloud Quickstart →](/docs/platform/quickstart/cloud-quickstart)
  </Tab>
  <Tab value="Self-Hosted">
    Deploy Fleetbase on your own cloud account (AWS, GCP, Azure, DigitalOcean) using Docker Compose or Kubernetes. You get full control over:

    - Where your data lives
    - Which version you run
    - What extensions are installed
    - Infrastructure scaling decisions

    [Deploy in Cloud →](/docs/platform/quickstart/deploy-in-cloud)
  </Tab>
  <Tab value="On-Premise">
    Run Fleetbase on your own hardware — in your data centre or on air-gapped infrastructure. Required services are MySQL, Redis, and S3-compatible object storage. No internet connection required after initial setup.

    [Run Locally →](/docs/platform/quickstart/running-locally)
  </Tab>
</Tabs>
