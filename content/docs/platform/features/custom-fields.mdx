# Custom Fields

Every logistics operation is unique. While Fleetbase provides comprehensive data models for common resources like Orders, Drivers, and Vehicles, you will inevitably need to capture data that is specific to your business.

**Custom Fields** allow you to add your own data attributes to any resource in Fleetbase without writing any code or modifying the underlying database schema.

## How Custom Fields Work

Under the hood, Fleetbase uses a flexible JSON-based metadata system to store custom fields. When you define a custom field for a resource (e.g., adding a "License Expiry Date" to a Driver), the Fleetbase Console automatically generates the appropriate UI input for that field whenever you create or edit a Driver.

### Supported Field Types

You can create custom fields of various types to ensure data integrity:

- **Text**: Short, single-line text inputs (e.g., "Gate Code").
- **Number**: Numeric inputs (e.g., "Pallet Count").
- **Date/Time**: Date pickers (e.g., "Insurance Expiry").
- **Boolean**: Checkboxes or toggles (e.g., "Requires Refrigeration").
- **Select**: Dropdown menus with predefined options (e.g., "Vehicle Type: Van, Truck, Bike").

## Creating a Custom Field

To add a custom field to a resource:

1. Navigate to **Settings** > **Custom Fields** in the Fleetbase Console.
2. Click **Create Custom Field**.
3. Select the **Target Resource** (e.g., `Order`, `Driver`, `Vehicle`).
4. Enter a **Name** for the field (e.g., "Special Instructions").
5. Select the **Field Type** (e.g., `Text`).
6. (Optional) Provide a **Default Value** or mark the field as **Required**.
7. Click **Save**.

Once saved, the new field will immediately appear in the creation and edit forms for the selected resource across the Fleetbase Console.

## Using Custom Fields via the API

Custom fields are fully supported by the Fleetbase REST API. When you retrieve a resource via the API, its custom fields are included in the `meta` object.

### Example: Retrieving an Order with Custom Fields

```json
{
  "id": "order_123",
  "public_id": "ORD-123",
  "status": "created",
  "meta": {
    "gate_code": "1234#",
    "requires_refrigeration": true,
    "pallet_count": 4
  }
}
```

### Example: Creating a Resource with Custom Fields

When creating or updating a resource via the API, you can pass custom fields by including them in the `meta` object of your request payload.

```bash
curl -X POST "https://api.fleetbase.io/fleet-ops/v1/orders" \
  -H "Authorization: Bearer flb_test_your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "pickup": "place_123",
    "dropoff": "place_456",
    "meta": {
      "gate_code": "1234#",
      "requires_refrigeration": true
    }
  }'
```

## Filtering and Searching

Custom fields are indexed, meaning you can use them to filter and search for resources in the Fleetbase Console. For example, if you added a "Vehicle Type" custom field to your Vehicles, you can easily filter your fleet list to show only "Refrigerated Trucks".
