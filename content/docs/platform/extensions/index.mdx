# Extensions

Fleetbase is built on a modular architecture. The core platform provides essential infrastructure—like identity management, API routing, and real-time events—but the actual logistics capabilities are provided by **Extensions**.

Extensions are modular packages that you can install to add specific features, workflows, or integrations to your Fleetbase instance. This architecture allows you to tailor the platform to your exact operational needs without bloating the system with features you don't use.

## How Extensions Work

An extension typically consists of two parts:

1. **Backend Package**: A PHP Laravel package that registers new API routes, database models, and business logic.
2. **Frontend Engine**: An Ember.js engine that registers new UI components, routes, and sidebar menus in the Fleetbase Console.

When an extension is installed, the Fleetbase Extension Bridge automatically wires these components into the core platform. The new API routes become available, and the new UI components appear in the console.

## Official Extensions

Fleetbase provides several official extensions that cover common logistics use cases:

- **[FleetOps](/docs/fleetops)**: A comprehensive fleet management and dispatch system. Manage drivers, vehicles, orders, and real-time tracking.
- **[Storefront](/docs/storefront)**: A headless e-commerce engine. Manage products, categories, customers, and checkout flows.
- **[Ledger](/docs/ledger)**: An accounting and billing module. Manage wallets, transactions, and invoicing.
- **[Pallet](/docs/pallet)** *(Beta)*: A warehouse and inventory management system.

## The Extension Registry

The **Fleetbase Registry** is the official marketplace for extensions. It hosts both official extensions built by the Fleetbase team and third-party extensions built by the community.

You can browse and install extensions directly from the Fleetbase Console or using the Fleetbase CLI.

- **[Browsing & Installing](/docs/platform/extensions/browsing-and-installing)**: Learn how to find and install extensions to your instance.
- **[Managing Extensions](/docs/platform/extensions/managing-extensions)**: Learn how to update, configure, or uninstall existing extensions.

## Building Custom Extensions

If the available extensions don't meet your specific needs, you can build your own. Fleetbase provides a comprehensive set of tools and APIs for developers to create custom extensions that integrate seamlessly with the core platform.

Custom extensions can be kept private for your organization's internal use, or published to the Fleetbase Registry to be shared with or sold to the community.

To learn more about building custom extensions, see the **[Building & Extending Fleetbase](/docs/building-and-extending)** section.
