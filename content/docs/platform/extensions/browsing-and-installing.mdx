# Browsing & Installing Extensions

Extensions are the building blocks of Fleetbase, adding specific logistics capabilities to the core platform. You can browse and install extensions from the Fleetbase Registry using either the Fleetbase Console UI or the Fleetbase CLI.

## Browsing Extensions

The Fleetbase Registry hosts both official extensions built by the Fleetbase team and third-party extensions built by the community.

### Via the Console UI

If you are using Fleetbase Cloud or have a self-hosted instance running, you can browse extensions directly from the web interface:

1. Log into the Fleetbase Console.
2. In the left sidebar, navigate to **Extensions**.
3. The marketplace will display featured, popular, and newly released extensions.
4. Use the search bar or category filters to find specific extensions (e.g., "fleet management", "accounting").

### Via the CLI

If you prefer the command line, you can search the registry using the Fleetbase CLI (`flb`):

```bash
# List all available extensions
flb search

# Search by keyword
flb search fleet

# Search by category
flb search --category logistics

# Filter for free extensions only
flb search --free

# Output results in JSON format (useful for scripting)
flb search --json
```

## Installing Extensions

The installation process differs slightly depending on whether you are using Fleetbase Cloud or a self-hosted instance.

### Installing on Fleetbase Cloud

For Fleetbase Cloud users, installing an extension is a one-click process:

1. Navigate to **Extensions** in the Fleetbase Console.
2. Find the extension you want to install (e.g., FleetOps).
3. Click the **Install** button.
4. The platform will automatically provision the extension for your organization. Once complete, the extension's menus and features will appear in your console.

### Installing on a Self-Hosted Instance

If you are self-hosting Fleetbase, you must use the Fleetbase CLI to install extensions. This ensures that the necessary backend packages and frontend engines are properly downloaded and wired into your infrastructure.

Before installing an extension, you must authenticate your CLI with the Fleetbase Registry:

1. **Register an account** (one-time setup):
   ```bash
   flb register
   ```
2. **Verify your email**:
   ```bash
   flb verify -e your-email@example.com -c verification-code
   ```
3. **Generate your registry token**:
   ```bash
   flb generate-token -e your-email@example.com
   ```
4. **Set your authentication token**:
   ```bash
   flb set-auth your-registry-token-here
   ```

Once authenticated, you can install extensions by their package name:

```bash
# Install the official FleetOps extension
flb install @fleetbase/fleetops

# Install the official Storefront extension
flb install @fleetbase/storefront
```

The CLI will automatically download the extension, run any necessary database migrations, and rebuild the frontend console to include the new UI components.

## Post-Installation

After an extension is installed, it may require additional configuration.

- **Permissions**: The extension will automatically register new permissions in the IAM system. You may need to update your [Roles](/docs/platform/iam/users-roles-permissions) to grant users access to the new features.
- **Settings**: Many extensions add new configuration panels under the **Settings** menu. Review these settings to tailor the extension to your operational needs.
