# Socket Events

Logistics operations require real-time visibility. Fleetbase uses **SocketCluster** to provide a highly scalable WebSocket server that broadcasts live updates across the platform.

By subscribing to socket events, you can build real-time dashboards, live-tracking interfaces for customers, or mobile apps that react instantly to changes in driver locations or order statuses.

## How Socket Events Work

When an event occurs in the Fleetbase backend (e.g., a driver updates their location via the Navigator app), the backend publishes the event to a Redis channel. The SocketCluster server subscribes to these channels and broadcasts the events to all connected WebSocket clients.

### Channels and Events

Socket communication in Fleetbase is organized around **Channels** and **Events**.

- **Channels**: Represent a specific resource or context (e.g., a specific order, a specific driver, or a company-wide channel).
- **Events**: Represent the specific action that occurred within that channel (e.g., `driver.location_updated`, `order.dispatched`).

## Connecting to the Socket Server

To receive real-time updates, your client application must connect to the Fleetbase SocketCluster server.

If you are building a web application, you can use the official `socketcluster-client` library.

```javascript
import socketClusterClient from 'socketcluster-client';

// Initialize the connection
const socket = socketClusterClient.create({
  hostname: 'socket.fleetbase.io', // Or your self-hosted socket domain
  secure: true,
  port: 443,
});

// Listen for connection events
socket.on('connect', () => {
  console.log('Connected to Fleetbase Socket Server');
});
```

## Subscribing to Channels

Once connected, you can subscribe to specific channels to receive events.

### Company Channel

The company channel broadcasts events relevant to your entire organization. The channel name is typically your organization's unique identifier (UUID).

```javascript
// Subscribe to the company channel
const companyChannel = socket.subscribe('company_uuid_here');

// Listen for specific events on this channel
companyChannel.on('driver.location_updated', (data) => {
  console.log('Driver moved:', data.driver_id, data.location);
  // Update map marker...
});

companyChannel.on('order.status_changed', (data) => {
  console.log('Order status updated:', data.order_id, data.status);
  // Update UI...
});
```

### Resource-Specific Channels

You can also subscribe to channels for specific resources, such as a single order. This is useful for building customer-facing tracking pages where the user only needs updates for their specific delivery.

```javascript
// Subscribe to a specific order's channel
const orderChannel = socket.subscribe('order_uuid_here');

orderChannel.on('order.dispatched', (data) => {
  console.log('Your order is on the way!');
});
```

## Monitoring Socket Events

The Fleetbase Console provides a built-in tool for monitoring real-time socket traffic.

1. Navigate to **Developers** > **Socket Events**.
2. The panel will display a live feed of all events being broadcasted on your organization's channel.
3. You can click on individual events to inspect the JSON payload.

This tool is invaluable for debugging real-time integrations and verifying that events are being fired correctly.

## Security Considerations

When deploying Fleetbase, the `SOCKETCLUSTER_OPTIONS` environment variable controls which domains are allowed to connect to the WebSocket server.

**Always restrict origins to your specific domains in production** to prevent unauthorized clients from connecting to your socket server and listening to events.

```yaml
# Example production configuration in docker-compose.override.yml
SOCKETCLUSTER_OPTIONS: '{"origins":"https://yourdomain.com:*,wss://yourdomain.com:*"}'
```
