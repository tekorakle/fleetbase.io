# API Keys

API keys are the primary method for authenticating requests to the Fleetbase REST API. Whether you are integrating an external ERP system, building a custom mobile app, or automating workflows, you will need an API key to securely interact with your Fleetbase instance.

## Creating an API Key

To generate a new API key:

1. Navigate to **Developers** > **API Keys** in the Fleetbase Console.
2. Click **Create API Key**.
3. Provide a descriptive **Name** for the key (e.g., "ERP Integration", "Mobile App Backend").
4. Select the **Environment** (Test or Live).
5. (Optional) Enter an **IP Whitelist** to restrict usage to specific IP addresses.
6. Click **Save**.

Once created, the API key will be displayed. **Copy this key and store it securely.** For security reasons, you will not be able to view the full key again after leaving the page.

## Authentication

Fleetbase uses Bearer token authentication. To authenticate an API request, include your API key in the `Authorization` header of your HTTP request.

```http
Authorization: Bearer flb_test_your_api_key_here
```

### Example Request

Here is an example of authenticating a request to retrieve a list of orders using cURL:

```bash
curl -X GET "https://api.fleetbase.io/fleet-ops/v1/orders" \
  -H "Authorization: Bearer flb_test_your_api_key_here" \
  -H "Content-Type: application/json"
```

## Test vs. Live Keys

Every API key is scoped to a specific environment, indicated by its prefix:

- **Test Keys** (`flb_test_...`): These keys operate on a separate, isolated dataset. Actions performed with a Test key do not affect your production data. Use Test keys during development, integration testing, and experimentation.
- **Live Keys** (`flb_live_...`): These keys operate on your production data. Use Live keys only for actual operations and production deployments.

When viewing the API Keys panel in the Developer Console, you can toggle between Test and Live modes using the switch in the top navigation bar to view the respective keys.

## Security Best Practices

To ensure the security of your Fleetbase instance, follow these best practices when managing API keys:

1. **Never share your API keys publicly.** Do not commit them to version control systems (like GitHub) or embed them in client-side code (like JavaScript or mobile apps) where they can be easily extracted.
2. **Use IP Whitelisting.** If your integration runs on a server with a static IP address, restrict the API key to that IP address. This prevents unauthorized use even if the key is compromised.
3. **Rotate keys regularly.** Periodically generate new API keys, update your integrations, and delete the old keys.
4. **Use descriptive names.** Clearly name your API keys so you can easily identify their purpose and revoke them if necessary.
5. **Monitor usage.** Regularly review the [Request Logs](/docs/platform/developer-console/request-logs) to monitor API traffic and detect any unusual activity.
