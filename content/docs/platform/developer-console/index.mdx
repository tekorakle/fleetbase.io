# Developer Console

The **Developer Console** is a built-in suite of tools designed to help you integrate Fleetbase with external systems, build custom applications, and monitor API traffic. It is accessible from the main sidebar in the Fleetbase Console under the **Developers** section.

Whether you are connecting your ERP system, building a custom mobile app, or setting up automated workflows, the Developer Console provides the necessary infrastructure.

## Core Features

The Developer Console is divided into four main areas:

### 1. [API Keys](/docs/platform/developer-console/api-keys)
Generate and manage the authentication tokens required to interact with the Fleetbase REST API. API keys can be scoped to specific environments (Test or Live) and restricted to specific IP addresses for enhanced security.

### 2. [Webhooks](/docs/platform/developer-console/webhooks)
Configure HTTP callbacks to receive real-time notifications when specific events occur within Fleetbase (e.g., when an order is created, or a driver's status changes). Webhooks are essential for keeping external systems in sync with your logistics operations.

### 3. [Socket Events](/docs/platform/developer-console/socket-events)
Monitor and subscribe to real-time WebSocket events broadcasted by the Fleetbase SocketCluster server. This is crucial for building live-tracking interfaces or real-time dashboards.

### 4. [Request Logs](/docs/platform/developer-console/request-logs)
View a complete, searchable history of all API requests made to your Fleetbase instance. Request logs include the endpoint, method, payload, response status, and the specific API key used, making it an invaluable tool for debugging integrations.

## Test vs. Live Environments

Fleetbase provides two distinct environments for your data: **Test** and **Live**.

- **Test Mode**: Used for development, integration testing, and experimentation. Data created in Test mode is completely isolated from your production data.
- **Live Mode**: Used for actual operations.

When using the Developer Console, you can toggle between Test and Live modes using the switch in the top navigation bar. This toggle affects which API keys, webhooks, and request logs are displayed. For example, if you are in Test mode, you will only see API keys generated for the Test environment, and the Request Logs will only show traffic generated by those Test keys.
