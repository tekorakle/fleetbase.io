# Quickstart

This guide walks you through creating your first order in Fleet-Ops, assigning it to a driver, and dispatching it to the Navigator app. It assumes Fleet-Ops is installed and you have at least one driver account set up.

## Prerequisites

- At least one driver created in **Fleet-Ops → Resources → Drivers**
- The Navigator app installed on the driver's phone (see [Navigator App Setup](/docs/fleet-ops/getting-started/navigator-app-setup))

## Step 1 — Open Fleet-Ops

Click **Fleet-Ops** in the top navigation bar of the Fleetbase Console. The Fleet-Ops dashboard opens showing the live map and the operations sidebar.

![Fleet-Ops main dashboard with live map and sidebar navigation](/images/placeholder.png)

## Step 2 — Create a New Order

<Steps>
  <Step>
    ### Open the Orders list

    In the left sidebar, click **Operations → Orders**. This shows all existing orders with their current status.
  </Step>
  <Step>
    ### Click New Order

    Click the **+ New Order** button in the top right. The new order form opens.

    ![New order form — order config selector, pickup and dropoff fields](/images/placeholder.png)
  </Step>
  <Step>
    ### Select an Order Configuration

    Choose an **Order Config** from the dropdown. The order config determines the workflow steps, entity fields, and proof of delivery method for this order. For a basic delivery, select the default **Delivery** config.
  </Step>
  <Step>
    ### Set the Pickup and Dropoff

    Enter the **Pickup** address (where the driver collects the item) and the **Dropoff** address (where it is delivered). You can type an address for geocoding, or select a saved Place.
  </Step>
  <Step>
    ### Add Entities (optional)

    Entities are the individual items being delivered. Click **Add Entity** to attach an item with a description, barcode, dimensions, and weight. Entities each get their own tracking number.
  </Step>
  <Step>
    ### Save the Order

    Click **Save**. The order is created in **pending** status and appears in the orders list.
  </Step>
</Steps>

## Step 3 — Assign a Driver

<Steps>
  <Step>
    ### Open the order detail

    Click the order row in the list to open the order detail panel.
  </Step>
  <Step>
    ### Assign driver and vehicle

    In the detail panel, click **Assign Driver**. Select a driver from the list of available drivers. Optionally assign a vehicle.

    The order status moves to **assigned**.
  </Step>
</Steps>

## Step 4 — Dispatch

<Steps>
  <Step>
    ### Dispatch the order

    With the order detail open, click **Dispatch**. This sends the order to the driver's Navigator app.

    The order status moves to **dispatched**.
  </Step>
  <Step>
    ### Driver accepts in Navigator app

    The driver receives a push notification in the Navigator app. When they accept and start the order, the status moves to **started** and their live location appears on the map.
  </Step>
</Steps>

## Step 5 — Track Progress

Open **Operations → Orders** and click the order, or click the driver's marker on the live map. The order detail shows:

- Current driver location and ETA
- Activity history (each status transition with timestamp)
- Proof of delivery when captured

![Order detail panel — driver location, ETA, activity timeline](/images/placeholder.png)

The order completes automatically when the driver captures proof of delivery (signature, photo, or QR scan) and marks the order as delivered in the Navigator app.
