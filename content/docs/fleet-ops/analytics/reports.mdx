# Reports

While the [Analytics](/docs/fleetops/analytics) dashboard provides a high-level, visual overview of your fleet's performance, the **Reports** engine allows you to dig deep into the raw data.

Reports are essential for auditing, compliance, payroll, and detailed operational analysis.

## Creating a Report

To generate a new report:

1. Navigate to **FleetOps** > **Reports** in the Fleetbase Console.
2. Click the **New Report** button.
3. Select the **Subject** of the report. This determines the primary data source (e.g., `Orders`, `Drivers`, `Vehicles`, `Work Orders`, `Fuel Reports`).
4. Define the **Columns** you want to include in the report (e.g., for an Orders report, you might select `Tracking Number`, `Status`, `Pickup Address`, `Dropoff Address`, `Total Distance`, `Total Cost`).
5. Apply **Filters** to narrow down the data (e.g., "Only show orders completed in the last 30 days", or "Only show orders assigned to the 'Downtown Fleet'").
6. Click **Run Report**.

FleetOps will query the database and display the results in a tabular format.

## Common Report Types

The flexibility of the Reports engine allows you to create a wide variety of analyses. Some common examples include:

### 1. Driver Payroll / Settlement Report
- **Subject**: Orders
- **Filters**: `Status` = `Completed`, `Date Range` = `Last Week`
- **Columns**: `Driver Name`, `Order Count`, `Total Distance Driven`, `Total Earnings` (if you pay drivers per order or per mile).
- **Use Case**: Calculating how much to pay your independent contractors or employees at the end of the week.

### 2. Vehicle Maintenance Cost Report
- **Subject**: Work Orders
- **Filters**: `Status` = `Completed`, `Date Range` = `Last Quarter`
- **Columns**: `Vehicle Name`, `Work Order Title`, `Total Labor Cost`, `Total Parts Cost`, `Total Cost`.
- **Use Case**: Identifying which vehicles are costing the most to maintain, helping you decide when it's time to retire an aging asset.

### 3. On-Time Performance Report
- **Subject**: Orders
- **Filters**: `Status` = `Completed`, `Date Range` = `Last Month`
- **Columns**: `Tracking Number`, `Customer Name`, `Scheduled ETA`, `Actual Completion Time`, `Variance` (the difference between ETA and actual).
- **Use Case**: Identifying bottlenecks in your routing or specific drivers who consistently miss their delivery windows.

### 4. Fuel Efficiency Report
- **Subject**: Fuel Reports
- **Filters**: `Date Range` = `Last Month`
- **Columns**: `Vehicle Name`, `Driver Name`, `Total Fuel Volume`, `Total Fuel Cost`, `Calculated MPG / L/100km`.
- **Use Case**: Spotting inefficient driving habits or mechanical issues that are causing excessive fuel consumption.

## Exporting Data

Once you have run a report and verified the data, you can export it for use outside of Fleetbase.

Click the **Export** button to download the report as a CSV (Comma Separated Values) file. This file can be opened in Microsoft Excel, Google Sheets, or imported into specialized Business Intelligence (BI) or accounting software.

## Saving Report Templates

If you find yourself running the same report frequently (e.g., a weekly payroll report), you can save the configuration as a template.

1. After configuring the columns and filters, click **Save as Template**.
2. Give the template a descriptive name.

The next time you need that data, you can simply select the template from the Reports list, adjust the date range if necessary, and run it instantly.
