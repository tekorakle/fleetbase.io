# Navigator App Settings

**Navigator App Settings** control how the Navigator driver mobile app behaves for your operation — login method, order acceptance mode, GPS tracking frequency, and driver onboarding configuration.

Navigate to **Fleet-Ops → Settings → Navigator App**.

![Navigator app settings panel — login method, acceptance mode, broadcast frequency, onboarding fields](/images/placeholder.png)

## Login Settings

| Setting | Options | Description |
| :--- | :--- | :--- |
| **SMS Verification** | On / Off | Require drivers to verify their phone number with an SMS code on each login |
| **Allow Driver Registration** | On / Off | Allow new drivers to self-register via the app without being pre-created in the console |
| **Registration Requires Approval** | On / Off | Hold self-registered drivers in pending status until an operator approves them |

<Callout type="info">
If **Allow Driver Registration** is off, drivers must be created manually in **Fleet-Ops → Resources → Drivers** before they can log in.
</Callout>

## Order Acceptance

| Setting | Options | Description |
| :--- | :--- | :--- |
| **Order Acceptance Mode** | Auto / Manual | **Auto** — orders are accepted automatically when dispatched. **Manual** — drivers must tap Accept in Navigator before the order starts. |
| **Accept Timeout** | Seconds | How long a manual-accept order waits before being automatically reassigned (0 = no timeout) |

## GPS Tracking

| Setting | Description |
| :--- | :--- |
| **Broadcast Frequency** | How often the driver's GPS position is sent to Fleet-Ops (seconds). Lower values = more real-time tracking but higher battery and data usage. Default: 5 seconds. |
| **Accuracy Mode** | **High Accuracy** (GPS + network) or **Battery Saver** (network only). High accuracy is recommended for urban operations. |
| **Background Tracking** | Must be enabled for continuous tracking when Navigator is in the background. Drivers must grant "Always Allow" location permission. |

## Driver Onboarding

Configure the onboarding flow shown to new drivers on first login:

| Setting | Description |
| :--- | :--- |
| **Welcome Message** | Custom message shown on the first screen after login |
| **Required Fields** | Fields the driver must complete during onboarding (e.g., license number, vehicle confirmation) |
| **Terms Acceptance** | Require driver to accept terms and conditions before proceeding |
| **Profile Photo** | Require driver to upload a profile photo during onboarding |

## Language and Locale

| Setting | Description |
| :--- | :--- |
| **Default Language** | The language shown in Navigator for all drivers in your organization |
| **Date Format** | Date display format (e.g., DD/MM/YYYY or MM/DD/YYYY) |

Drivers can override the language in their individual app settings if the app supports multiple languages.

## Notification Preferences

| Setting | Description |
| :--- | :--- |
| **Push Notifications** | Enable/disable push notifications to the Navigator app for dispatches and updates |
| **Sound Alerts** | Play a sound when a new order arrives |
| **Vibration** | Vibrate on incoming order notification |

## Saving Settings

Click **Save** to apply changes. Navigator app settings take effect for all drivers immediately — the next time Navigator connects to the server, it picks up the updated configuration.
