# Navigator App Settings

**Navigator App Settings** control two operation-specific behaviours of the Navigator driver mobile app: which entity fields drivers are allowed to edit per Order Configuration, and whether drivers can onboard themselves from the app.

Navigate to **Fleet-Ops → Settings → Navigator App**.

![Navigator app settings panel — entity field editing settings and driver onboard settings](/images/screenshots/fleet-ops/fleet-ops-navigator-app-settings.webp)

## Entity Field Editing

The **Entity Field Editing Settings** panel controls which fields on a payload's entities the driver can edit from inside the Navigator app, scoped per [Order Configuration](/docs/fleet-ops/operations/order-configurations/overview).

<Steps>
  <Step>
    Pick the Order Configuration you want to configure from the **Select order config** dropdown.
  </Step>
  <Step>
    Toggle **Enable driver to edit entity fields**. When off, all entity fields are read-only in Navigator for orders of this config.
  </Step>
  <Step>
    When the toggle is on, check the individual fields drivers should be able to edit (description, weight, dimensions, custom fields, etc.).
  </Step>
  <Step>
    Click **Save changes**. The new setting applies the next time a driver opens an order of this config in Navigator.
  </Step>
</Steps>

The setting is stored per Order Configuration so different order types can grant different levels of driver autonomy — for example a courier flow might lock everything, while a maintenance flow lets drivers update notes and weights inline.

## Driver Onboarding from the App

The **Driver Onboard Settings** panel controls whether new drivers can register themselves from inside the Navigator app, without being pre-created by an operator in the console.

| Setting | Description |
| :--- | :--- |
| **Enable driver onboard from app** | Master toggle — when on, the Navigator login screen exposes a *Sign up* flow |
| **Select onboard method** | The verification path used during sign-up |
| **Require driver to upload onboard documents** | When on, the sign-up flow blocks completion until the driver uploads the documents you require |
| **Required onboard documents** | Free-text list of document names the driver must upload (e.g. *Driver's License*, *Insurance Certificate*) |

When self-registration is off, drivers must exist in **Fleet-Ops → Resources → Drivers** before they can log in.

<Callout type="info">
Other Navigator app behaviours — GPS broadcast frequency, push notifications, locale, and order acceptance mode — are managed by the platform's general system-setup pages or by the driver's device permissions, not by these Fleet-Ops Navigator app settings.
</Callout>

## Saving Settings

Each panel has its own **Save changes** button. Updates apply the next time Navigator connects to the server.
