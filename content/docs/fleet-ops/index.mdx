# Fleet-Ops

Fleet-Ops is the core logistics and fleet management extension for the Fleetbase platform. It gives operations teams a full dispatch console for managing orders, drivers, vehicles, and routes — plus a mobile driver app (Navigator) that handles real-time GPS tracking, turn-by-turn navigation, and proof of delivery capture.

![Fleet-Ops dashboard — live map with active driver positions and order status panel](/images/placeholder.png)

## What Fleet-Ops Does

Fleet-Ops covers the full delivery and fleet operations lifecycle in one place:

| Capability | What it covers |
| :--- | :--- |
| **Order Management** | Create, assign, dispatch, and track deliveries end to end |
| **Live Map** | Real-time driver and vehicle positions with route overlays |
| **Driver App (Navigator)** | Mobile app for drivers — navigation, status updates, POD capture |
| **Fleet & Driver Management** | Organize drivers and vehicles into fleets, track availability and status |
| **Orchestrator** | Automated batch order assignment with route optimization |
| **Scheduler** | Driver shift scheduling and order timeline management |
| **Order Configurations** | Custom order types with configurable workflows and entity fields |
| **Service Rates & Quotes** | Pricing rules and on-demand quotation for orders |
| **Maintenance** | Preventive schedules, work orders, and parts inventory |
| **Telematics & Connectivity** | Device, sensor, and telematics modem integration |
| **Analytics** | Fleet performance dashboards and custom reports |

## Key Concepts

Fleet-Ops is organized around a few core entities:

- **Order** — a delivery or service task with a pickup location, a destination, and items (entities). Orders move through a configurable status lifecycle and are captured with proof of delivery.
- **Driver** — a person who executes orders using the Navigator app. Drivers have real-time location, shift schedules, and skill constraints that the Orchestrator uses for assignment.
- **Vehicle** — a fleet asset with capacity, location tracking, and maintenance history.
- **Fleet** — a logical group of drivers and vehicles, typically scoped to a route, region, or service type.
- **Order Configuration** — a template that defines a custom order type: its activity flow, the entity fields required, and the proof of delivery method.

See [Core Concepts](/docs/fleet-ops/getting-started/core-concepts) for a full introduction.
