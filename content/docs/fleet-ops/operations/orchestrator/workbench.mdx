# Orchestrator Workbench

The **Workbench** is the main UI for running the Orchestrator. It provides a side-by-side view of the unassigned order pool and the available driver/vehicle resources, with controls to run optimization, review the proposed plan, and commit the final assignments.

![Orchestrator Workbench — order pool panel, resource panel, and proposed assignment results](/images/placeholder.png)

## Opening the Workbench

Navigate to **Fleet-Ops → Operations → Orchestrator**. The Workbench opens showing:

- **Order Pool** (left panel) — all unassigned pending orders
- **Resource Panel** (right panel) — available drivers and vehicles
- **Results Panel** (bottom) — proposed assignments after running optimization

## Order Pool

The order pool lists all orders eligible for assignment. Each order card shows:

- Order ID and customer
- Pickup and dropoff addresses
- Priority level (if set)
- Required skills (if set)
- Time window (if set)
- Estimated distance

### Filtering the Pool

Use the filters above the pool to narrow which orders are included in the optimization run:

- Filter by **Fleet** to run the Orchestrator for a specific fleet's orders only
- Filter by **Date** to include only today's or tomorrow's orders
- Filter by **Order Config** to optimize one service type at a time

### Excluding Orders

Click the **X** on any order card to temporarily exclude it from the current run. Excluded orders remain pending and appear in the next run.

## Resource Panel

The resource panel shows available drivers and vehicles. Each driver card shows:

- Driver name, photo, and online status
- Current location on mini-map
- Skills (certifications)
- Shift window (time window start/end)
- Current number of assigned orders
- Vehicle assigned

Toggle drivers on/off to include or exclude them from the optimization run.

## Running Optimization

<Steps>
  <Step>
    ### Select the optimization engine

    Use the engine selector to choose: **VROOM** (default, recommended for most cases), **Greedy**, **Driver Assignment**, or **Route Sequencing**.
  </Step>
  <Step>
    ### Set parameters (optional)

    Adjust optimization parameters:
    - **Max orders per driver** — cap the workload per driver
    - **Balance workload** — try to equalize orders across drivers
    - **Geometry type** — Simple (straight lines) or Full Routes (uses routing engine for accurate ETAs)
  </Step>
  <Step>
    ### Click Run Optimization

    The engine processes the order pool against available resources. For large batches this may take a few seconds.
  </Step>
  <Step>
    ### Review the proposed plan

    The results panel shows proposed assignments:
    - Each driver row shows which orders they've been assigned
    - Total distance and estimated time per driver
    - Orders flagged as unassigned (with reasons)
    
    Click any assignment to see the proposed route on the map.
  </Step>
  <Step>
    ### Adjust the plan

    Before committing, you can:
    - **Drag orders** between driver rows to manually reassign
    - **Exclude orders** from the plan to leave them pending
    - **Re-run** with different parameters
  </Step>
  <Step>
    ### Commit

    Click **Commit Plan**. The Orchestrator:
    - Creates a **Manifest** for each driver with their ordered stop list
    - Assigns the driver and vehicle to each order
    - Dispatches orders to the Navigator app
    
    A confirmation shows the number of orders assigned and any that could not be allocated.
  </Step>
</Steps>

## Reviewing Results After Commit

After committing, each driver's manifest appears in:

- **Fleet-Ops → Resources → Drivers → [Driver] → Schedule** — the driver's order list for the day
- **Navigator app** — drivers see their manifest as their active job queue

Unassigned orders return to the pending pool and are available for the next Orchestrator run or manual assignment.

## Phase Builder

For complex multi-phase operations (e.g., warehouse pickup first, then delivery routes), the Workbench includes a **Phase Builder** that lets you define optimization phases. Each phase runs a separate optimization pass — useful when you want to first consolidate pickups, then plan delivery routes.

Access the Phase Builder using the **Phases** tab in the Workbench toolbar.
