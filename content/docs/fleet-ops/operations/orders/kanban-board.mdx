# Kanban Board

The Kanban Board provides a visual, column-based view of all orders organized by status. Each column represents a status stage — pending, assigned, dispatched, started, completed — and each card represents an order. It gives dispatchers a real-time snapshot of the operation's current state without navigating individual order records.

![Fleet-Ops Kanban board — columns for each order status, order cards with driver and address info](/images/placeholder.png)

## Accessing the Kanban Board

In Fleet-Ops, navigate to **Operations → Orders** and switch the view toggle from **List** to **Board** (Kanban view) using the view switcher in the top right of the orders page.

## Board Columns

The board has one column per order status:

| Column | Orders shown |
| :--- | :--- |
| **Pending** | Awaiting driver assignment |
| **Scheduled** | Assigned a future pickup date |
| **Assigned** | Driver assigned, not yet dispatched |
| **Dispatched** | Sent to Navigator app, awaiting driver start |
| **Started** | Driver actively executing the order |
| **Completed** | Delivered and closed |
| **Cancelled** | Cancelled orders |

By default, completed and cancelled orders are hidden to keep the board focused on active work. Toggle their visibility using the column filter.

## Order Cards

Each order card shows:

- **Order ID** (public_id — e.g., `ORD-00123`)
- **Customer** name
- **Dropoff address** summary
- **Assigned driver** name (if assigned)
- **Scheduled time** (if scheduled)
- **Status badge**

Click a card to open the full order detail panel.

## Filtering the Board

Use the filter bar at the top to narrow the board to:

- **Driver** — show only orders assigned to a specific driver
- **Fleet** — show only orders belonging to a fleet
- **Date** — filter by scheduled date or creation date
- **Order Config** — filter by order type

Filters stack — combine driver and date to see a specific driver's schedule for a given day.

## Quick Actions from Cards

Hover over an order card to reveal quick action buttons:

- **Dispatch** — dispatch an assigned order directly from the card without opening the detail
- **Assign** — open the assignment dialog for a pending order
- **Cancel** — cancel the order

## Using the Board with the Orchestrator

The Kanban Board and the [Orchestrator](/docs/fleet-ops/operations/orchestrator/overview) work well together. Use the Orchestrator to do bulk assignment and route optimization, then switch to the board view to monitor real-time progress as drivers work through their manifests.
