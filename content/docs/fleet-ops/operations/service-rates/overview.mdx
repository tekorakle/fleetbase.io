# Service Rates

**Service Rates** define the pricing rules Fleet-Ops uses to calculate the cost of an order. Rates can be flat fees, distance-based, weight-based, or zone-based. They support surcharges for COD (cash on delivery), peak hours, and parcel size. When an order matches a service rate, Fleet-Ops generates a **Service Quote** with the calculated amount.

## Rate Calculation Methods

The `rate_calculation_method` field on a service rate accepts one of six values. Each has its own deep-dive page covering formula, configuration fields, worked examples, and limitations:

| Value | Method | How it works |
| :--- | :--- | :--- |
| `per_meter` | [Per Meter](/docs/fleet-ops/operations/service-rates/per-meter) | Linear per-unit-of-distance pricing — `per_meter_flat_rate_fee` × distance in the selected unit |
| `fixed_meter` | [Fixed Rate](/docs/fleet-ops/operations/service-rates/fixed-rate) | Banded distance pricing — auto-generated `ServiceRateFee` rows, one per integer unit, each with its own flat fee. `fixed_rate` is accepted as a legacy alias. |
| `per_drop` | [Per Drop-off](/docs/fleet-ops/operations/service-rates/per-drop) | Tiered by stop count — `ServiceRateFee` rows with `min`/`max` ranges and a flat fee per range |
| `multi_zone_distance` | [Multi-zone Distance](/docs/fleet-ops/operations/service-rates/multi-zone-distance) | Geography-aware per-distance pricing — one rate per Zone or Service Area, plus an optional fallback for unmatched route segments |
| `parcel` | [Parcel Rate](/docs/fleet-ops/operations/service-rates/parcel) | Per-parcel pricing — `ServiceRateParcelFee` size tiers matched against each parcel's dimensions and weight |
| `algo` (alias: `algorithm`) | [Algorithm Rates](/docs/fleet-ops/operations/service-rates/algorithm-rates) | Custom math expression resolved at quote time, combining distance, time, stops, parcels, and other variables |

Zone-based and order-config-based pricing are **scopes** that apply to *any* method — see [Rate Scoping](#rate-scoping) below. (Note: this is distinct from `multi_zone_distance`, which uses zones as a pricing input rather than as a scope.)

## Rate Components

Each service rate has the following configurable fields:

### Base Fee
The starting charge applied before any distance or weight calculation.

### Distance Component
- **Per-meter rate** — charge per unit of distance
- **Per-meter unit** — km, mile, meter, or feet
- **Max distance** — cap beyond which no additional distance charge is added

### Cash on Delivery (COD) Fee
Applied when the order has a COD amount to collect from the recipient:
- **Flat COD fee** — fixed fee per COD order
- **COD percentage** — percentage of the COD amount

### Peak Hours Surcharge
Additional charge during high-demand periods:
- **Start/end time** — when peak hours apply
- **Flat peak fee** or **peak percentage** — the surcharge amount

### Banded Distance Fees (`fixed_meter`)
For [Fixed Rate](/docs/fleet-ops/operations/service-rates/fixed-rate) rates, **`ServiceRateFee`** entries define the fee table. Fleet-Ops auto-generates one row per integer unit of distance up to `max_distance`, and each row has its own flat fee. The lookup picks the row whose band the order's distance falls into.

### Per-Drop Tiered Fees (`per_drop`)
For [Per Drop-off](/docs/fleet-ops/operations/service-rates/per-drop) rates, **`ServiceRateFee`** entries define a tier table with `min`/`max` ranges. Each row's fee is charged when the order's stop count falls into that range.

### Geographic Rules (`multi_zone_distance`)
For [Multi-zone Distance](/docs/fleet-ops/operations/service-rates/multi-zone-distance) rates, **`ServiceRateFee`** entries define geographic rules. Each row references a Zone (`zone_uuid`) or Service Area (`service_area_uuid`), has a per-distance `fee` in its own `distance_unit`, and supports `priority` (for overlap resolution) and `is_fallback` (for unmatched route segments).

### Parcel Fees (`parcel`)
For [Parcel Rate](/docs/fleet-ops/operations/service-rates/parcel) rates, **`ServiceRateParcelFee`** entries define a fee table keyed by parcel attributes — `size`, `length`/`width`/`height`/`dimensions_unit`, and `weight`/`weight_unit`. Each entry has a `fee` and `currency`. The lookup picks the entry that matches the parcel's measurements.

## Creating a Service Rate

<Steps>
  <Step>
    Navigate to **Fleet-Ops → Operations → Service Rates**.
  </Step>
  <Step>
    Click **+ New Rate**.
  </Step>
  <Step>
    Set the **Service Name** (free-text label, e.g. *Same-Day Express*) and **Service Type** (free-text grouping, e.g. *delivery*, *transport*, *parcel*). Service type is not an enum — use whatever taxonomy fits your operation.
  </Step>
  <Step>
    Select the **Rate Calculation Method** from the options above.
  </Step>
  <Step>
    Set the **Base Fee** and configure the distance/weight/volume components as needed.
  </Step>
  <Step>
    Set the **Currency** for this rate.
  </Step>
  <Step>
    Configure optional surcharges: COD fee, peak hours, parcel fees.
  </Step>
  <Step>
    Set the rate **Scope**:
    - **Service Area** — applies only to orders within a specific service area
    - **Zone** — applies only to orders within a specific zone
    - **Order Config** — applies only to a specific order type
  </Step>
  <Step>
    Click **Save**. The rate is now active and will be matched to new orders.
  </Step>
</Steps>

## Rate Scoping

Rates are matched to orders based on their scope. Multiple rates can exist for the same service type — the most specific match wins:

1. Zone-scoped rate (most specific)
2. Service area-scoped rate
3. Order config-scoped rate
4. Global rate (no scope)

If multiple rates match, the customer can choose which to apply, or the first match is used automatically depending on your settings.

## Duration Terms

Rates can include **Duration Terms** — the expected delivery timeframe for this service type (e.g., "Same Day", "Next Business Day", "2–3 Days"). This is shown on service quotes as the estimated delivery window.

## Via the API

Service rates can be read via the public API:

```bash
GET /v1/service-rates
```

Filtered by service area, zone, or order config to retrieve applicable rates for a given origin/destination pair.
