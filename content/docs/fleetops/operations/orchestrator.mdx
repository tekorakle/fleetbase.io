# Orchestrator

As your logistics network scales, manually dispatching every order becomes impossible. The **Orchestrator** is a powerful automation engine built into FleetOps that allows you to define rules for how orders should be handled automatically.

Think of the Orchestrator as a set of "If This, Then That" (IFTTT) rules for your dispatch operations.

## How the Orchestrator Works

The Orchestrator evaluates every new order against a set of rules you define. If an order matches the conditions of a rule, the Orchestrator automatically executes the associated actions.

A rule consists of two parts:
1. **Conditions**: The criteria that must be met for the rule to trigger.
2. **Actions**: What happens when the rule is triggered.

## Creating an Orchestrator Rule

To set up a new automation rule:

1. Navigate to **FleetOps** > **Orchestrator** in the Fleetbase Console.
2. Click the **New Rule** button.
3. Provide a **Name** for the rule (e.g., "Auto-Assign Downtown Orders").
4. Define the **Conditions**.
5. Define the **Actions**.
6. Click **Save**.

## Common Conditions

You can trigger rules based on almost any attribute of an order:

- **Geographic Zones**: "If the pickup location is within the 'Downtown' Zone..."
- **Order Configuration**: "If the order uses the 'Heavy Freight' configuration..."
- **Payload Attributes**: "If the total weight is greater than 500kg..."
- **Customer**: "If the customer is 'Acme Corp'..."
- **Time**: "If the order is created between 10:00 PM and 6:00 AM..."

You can combine multiple conditions using AND/OR logic to create highly specific triggers.

## Common Actions

When a rule is triggered, the Orchestrator can perform several actions automatically:

- **Assign to a Fleet**: Automatically route the order to a specific regional or specialized fleet.
- **Assign to a Vendor**: Automatically overflow the order to a third-party [Integrated Vendor](/docs/fleetops/connectivity/integrated-vendors) (like Lalamove) if your internal fleet is at capacity or if the order is outside your normal service area.
- **Change Status**: Automatically mark the order as `dispatched` or `canceled`.
- **Send Notification**: Trigger a specific email or SMS alert to the customer or a manager.

## Example Use Cases

### 1. Regional Dispatching
If you operate in multiple cities, you can create rules that automatically assign orders to the correct regional fleet based on the pickup location's Zone.
- *Condition*: Pickup is in "London Zone".
- *Action*: Assign to "London Fleet".

### 2. Specialized Vehicle Routing
If an order requires a refrigerated truck, you can ensure it is only assigned to the fleet that has those vehicles.
- *Condition*: Order Configuration is "Cold Chain Delivery".
- *Action*: Assign to "Refrigerated Fleet".

### 3. Automated Overflow (3PL Integration)
If you receive an order outside of your normal operating hours, you can automatically route it to a third-party vendor.
- *Condition*: Order created between 22:00 and 06:00.
- *Action*: Assign to Vendor "NightOwl Couriers".

## Rule Priority

If an order matches the conditions for multiple rules, the Orchestrator executes them in order of priority. You can drag and drop rules in the Orchestrator list to set their priority, ensuring that more specific rules (e.g., "Acme Corp Orders in Downtown") are evaluated before more general rules (e.g., "All Downtown Orders").
