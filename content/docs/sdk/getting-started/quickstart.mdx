# Quickstart

The easiest way to start building a new Fleetbase extension is to use the **Fleetbase CLI**.

The CLI provides a `scaffold` command that generates a complete, working extension skeleton, including both the [Backend](/docs/sdk/backend/service-provider) (Laravel package) and the [Frontend](/docs/sdk/frontend/routing) (Ember Engine), along with the necessary `extension.json` manifest.

## Prerequisites

Before you begin, ensure you have the following installed on your local development machine:

- **Node.js** (v18 or higher)
- **PHP** (v8.2 or higher)
- **Composer**
- **Git**
- The **Fleetbase CLI**

If you haven't installed the CLI yet, you can do so via npm:

```bash
npm install -g @fleetbase/cli
```

## Scaffolding the Extension

1. Open your terminal and navigate to the directory where you want to create your extension.
2. Run the `scaffold` command:

```bash
flb scaffold my-extension
```

3. The CLI will prompt you for a few details:
   - **Extension Name**: The human-readable name of your extension (e.g., "My Custom Extension").
   - **Description**: A brief description of what the extension does.
   - **Author**: Your name or your organization's name.
   - **Namespace**: The PHP namespace for your backend package (e.g., `MyOrg\MyExtension`).

4. The CLI will then generate the directory structure and install the necessary dependencies.

## The Generated Structure

Once the scaffolding is complete, you will have a new directory called `my-extension` containing the following structure:

```text
my-extension/
├── addon/                 # The Ember.js frontend engine
├── server/                # The PHP Laravel backend package
├── extension.json         # The extension manifest
└── README.md
```

This is a fully functional, albeit empty, extension. It is ready to be linked to your local Fleetbase instance for development.

## Linking for Development

To develop your extension, you need to link it to a running Fleetbase instance. This allows you to see your changes in real-time without having to publish and install the extension via the Registry.

1. Ensure you have a local Fleetbase instance running (see [Quickstart (Self-Hosted)](/docs/getting-started/quickstart-self-hosted)).
2. Navigate into your newly scaffolded extension directory:

```bash
cd my-extension
```

3. Run the `install` command, providing the path to your local Fleetbase installation:

```bash
flb install
```

## Starting the Development Server

With the extension linked, you can now start the Fleetbase development servers.

1. Navigate to your local Fleetbase installation directory.
2. Start the backend server:

```bash
php artisan serve
```

3. In a separate terminal window, start the frontend server:

```bash
npm run start
```

4. Open your browser and navigate to `http://localhost:4200`.

You should now see your extension listed in the Fleetbase Console sidebar. Clicking on it will load the default route defined in your scaffolded frontend engine.

You are now ready to begin [Backend Development](/docs/sdk/backend/service-provider) and [Frontend Development](/docs/sdk/frontend/routing).
