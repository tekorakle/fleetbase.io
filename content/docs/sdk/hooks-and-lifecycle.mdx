# Hooks & Lifecycle Events

Fleetbase provides a robust **Hooks** system that allows your extension to tap into key lifecycle events across the platform.

This is particularly useful when you need to run custom logic in response to actions that occur outside of your extension's direct control (e.g., when a user logs in, when an order is created, or when a dashboard loads).

## The Hook Service

In your frontend Ember Engine, you interact with hooks using the `hook` service (which is often injected alongside the `universe` service).

### Registering a Hook

To listen for a specific event, you use the `on` method.

```javascript
import Route from '@ember/routing/route';
import { inject as service } from '@ember/service';

export default class ApplicationRoute extends Route {
    @service hook;

    init() {
        super.init(...arguments);

        this.hook.on('user.login', this.handleUserLogin.bind(this));
    }

    handleUserLogin(user) {
        console.log(`User ${user.name} logged in!`);
        // Run custom logic, fetch extension-specific data, etc.
    }
}
```

### Triggering a Hook

If your extension performs actions that other parts of the system might care about, you can trigger your own custom hooks using the `trigger` method.

```javascript
// Inside a component or controller
this.hook.trigger('my-extension.custom-event', { data: 'some payload' });
```

## Common Lifecycle Hooks

While you can define custom hooks, Fleetbase core and its official modules (like FleetOps) emit several standard hooks that you can listen to:

- **`user.login`**: Fired when a user successfully authenticates.
- **`user.logout`**: Fired when a user logs out.
- **`organization.changed`**: Fired when a user switches their active organization context.
- **`dashboard.loaded`**: Fired when the main Fleetbase dashboard finishes loading.
- **`fleetops.order.created`**: Fired when a new order is created in FleetOps.
- **`fleetops.order.dispatched`**: Fired when an order is assigned to a driver.

## Backend Webhooks

The frontend Hook service is for UI-level events. If you need to run server-side logic in response to platform events (e.g., sending an email when an order is completed, or syncing data with an external ERP), you should use **Webhooks**.

Fleetbase's [Developer Console](/docs/platform/developer-console/webhooks) allows you to configure webhooks that send HTTP POST requests to your extension's backend API whenever specific events occur in the database.

For example, you could configure a webhook for the `order.completed` event to hit an endpoint in your [Backend Package](/docs/extending-fleetbase/backend-development), allowing your Laravel controllers to handle the business logic securely.
