# Payments & Transactions

The **Payments** module covers three interconnected areas: payment gateways for processing real money, wallets for tracking internal balances, and transactions as the immutable audit trail of every financial event.

Navigate to **Ledger → Payments** to access gateways, wallets, and transactions.

![Payments overview — wallet balance summary cards, recent transaction feed](/images/placeholder.png)

## Core Components

### [Payment Gateways](/docs/ledger/payments/gateways)

External payment processors — Stripe, QPay, and Cash — integrated with encrypted credentials and sandbox support. Gateways process charges, refunds, and top-ups. Each gateway interaction is logged as a `GatewayTransaction` for idempotency and audit.

### [Wallets](/docs/ledger/payments/wallets)

Digital balance accounts tied to drivers, customers, vendors, or your company. Every wallet operation (deposit, withdrawal, transfer) creates a journal entry maintaining double-entry integrity. Wallets can be frozen to block debits.

### [Transactions](/docs/ledger/payments/transactions)

The complete, immutable ledger of every financial event. Transactions are never edited or deleted — they are the full audit trail for compliance and reconciliation.

## The Payment Lifecycle in a Marketplace

```
Customer pays via Stripe
  → GatewayTransaction logged (idempotency check)
    → PaymentSucceeded event fires
      → Ledger Transaction created (credit direction)
        → Journal Entry: DEBIT Cash, CREDIT Wallet Liability
          → Organization Wallet credited
            → Commission Transfer: DEBIT Org Wallet, CREDIT Merchant Wallet
              → Driver Earning: DEBIT Expense, CREDIT Driver Wallet
                → Payout: DEBIT Driver Wallet, CREDIT Cash (bank transfer)
```

## Webhook Security

All payment gateways send webhooks to confirm transactions. Ledger verifies webhook signatures before processing. Duplicate webhooks are deduplicated using `GatewayTransaction.alreadyProcessed()` — a check on `(gateway_reference_id, event_type)` — so each event is processed exactly once.

## Multi-Currency

Wallets are denominated in a single currency. If you operate in multiple currencies, create separate wallets per currency. Transaction records include `exchange_rate`, `settled_currency`, and `settled_amount` fields for multi-currency reconciliation.
