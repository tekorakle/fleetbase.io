# Transactions

A **Transaction** is the immutable record of money moving into, out of, or between wallets. Transactions are never edited or deleted — they are the complete, permanent audit trail for every financial event in your system.

Navigate to **Ledger → Payments → Transactions**.

![Transaction list — date, type, direction badge, amount, status, payer, payee](/images/placeholder.png)

## Transaction Attributes

| Field | Description |
| :--- | :--- |
| **Type** | What kind of financial event this is (see types below) |
| **Direction** | `credit` (money in) or `debit` (money out) relative to the wallet |
| **Status** | Current state of the transaction |
| **Amount** | Transaction amount |
| **Fee Amount** | Gateway or processing fee |
| **Tax Amount** | Tax applied to the transaction |
| **Net Amount** | Amount − Fees − Tax |
| **Balance After** | Wallet balance immediately after this transaction |
| **Currency** | Transaction currency |
| **Payer** | Who made the payment |
| **Payee** | Who received the payment |
| **Gateway** | Which gateway processed this (if applicable) |
| **Reference** | Human-readable reference (invoice number, order ID) |
| **Description** | What the transaction is for |

## Transaction Types

| Type | Description |
| :--- | :--- |
| `deposit` | Direct credit to a wallet (manual) |
| `withdrawal` | Direct debit from a wallet (manual payout) |
| `transfer` | Movement between two wallets |
| `payment` | Credit from a gateway charge (customer paying) |
| `refund` | Debit from a gateway refund (returning money to customer) |
| `topup` | Credit to a wallet via gateway (wallet top-up) |
| `payout` | Debit from a wallet to an external bank account |
| `earning` | Credit for completed work (e.g., driver delivery earning) |
| `fee` | Debit for platform or processing fees |

## Transaction Statuses

| Status | Meaning |
| :--- | :--- |
| `pending` | Initiated but not yet confirmed |
| `completed` | Successful; wallet balance updated |
| `failed` | Could not complete (declined card, insufficient funds) |
| `reversed` | Cancelled after completion; balance restored |
| `voided` | Invalidated before completion |
| `expired` | Expired before processing was confirmed |

## Filtering Transactions

Use the filter panel to narrow the list:

- **Direction** — credit or debit
- **Type** — filter by transaction type
- **Status** — filter by status
- **Gateway** — filter by payment processor
- **Date range** — specify start and end dates
- **Wallet** — show transactions for a specific wallet owner

## Transaction Detail

Click any transaction to see its full detail panel, including:
- Complete amount breakdown (fee, tax, net)
- Linked journal entry
- Gateway reference ID (for gateway transactions)
- Linked invoice or order (context)
- Failure reason (for failed transactions)

## Automated Transactions

Most transactions are created automatically:
- **Fleet-Ops order complete** → driver earning credit transaction
- **Storefront checkout** → customer payment transaction
- **Invoice paid via portal** → payment transaction
- **Wallet top-up** → deposit transaction

Manual transactions can be created via wallet operations (deposit, withdraw, transfer) from the wallet detail panel.
