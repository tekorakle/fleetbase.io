# Set Up Multi-Tenant Organizations

Fleetbase is designed from the ground up to support multi-tenancy. This means you can run a single instance of Fleetbase and host multiple independent organizations (tenants) on it, each with their own isolated data, users, and settings.

This guide explains how to configure Fleetbase for multi-tenant use cases using Organizations, Roles, and Permission Policies.

## Step 1: Create an Organization

The first step is to create a new Organization for your tenant.

You can do this via the Fleetbase Console (in the IAM section) or via the API:

```bash
curl -X POST https://api.fleetbase.io/v1/iam/organizations \
  -H "Authorization: Bearer flb_test_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Acme Logistics",
    "description": "A third-party logistics provider."
  }'
```

## Step 2: Create a Role

Next, create a Role for the users within this organization. Roles define what actions a user can perform.

```bash
curl -X POST https://api.fleetbase.io/v1/iam/roles \
  -H "Authorization: Bearer flb_test_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Dispatcher",
    "description": "Can manage orders and dispatch drivers.",
    "organization": "org_xxxxxxxx"
  }'
```

## Step 3: Assign Policies to the Role

Now, assign Permission Policies to the Role you just created. Policies are granular rules that grant or deny access to specific resources.

For example, to allow the Dispatcher role to manage orders, you would assign the `fleet-ops.order.*` policy.

```bash
curl -X POST https://api.fleetbase.io/v1/iam/roles/role_xxxxxxxx/policies \
  -H "Authorization: Bearer flb_test_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "policies": [
      "fleet-ops.order.*",
      "fleet-ops.driver.read"
    ]
  }'
```

## Step 4: Invite Users

Finally, invite users to join the organization and assign them the appropriate role.

```bash
curl -X POST https://api.fleetbase.io/v1/iam/users/invite \
  -H "Authorization: Bearer flb_test_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "dispatcher@acmelogistics.com",
    "organization": "org_xxxxxxxx",
    "role": "role_xxxxxxxx"
  }'
```

The user will receive an email invitation with a link to set their password and log in to the Fleetbase Console. Once logged in, they will only see data belonging to their organization.

## Data Isolation

Fleetbase automatically enforces data isolation between organizations. When a user makes an API request or views the console, Fleetbase automatically scopes the query to their current organization.

You do not need to manually filter queries by organization ID in your extension code; the `CompanyScope` global scope handles this automatically for all models that implement the `HasCompanyScope` trait.
