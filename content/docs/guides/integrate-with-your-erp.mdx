# Integrate Fleetbase with Your ERP

This guide explains how to integrate Fleetbase with your existing Enterprise Resource Planning (ERP) system, such as SAP, Oracle, NetSuite, or Microsoft Dynamics.

By integrating Fleetbase with your ERP, you can:
1. Automatically create delivery orders in Fleetbase when a sales order is created in your ERP.
2. Sync inventory levels between your ERP and Fleetbase [Pallet](/docs/pallet/overview).
3. Update the ERP with real-time delivery statuses and proof of delivery from Fleetbase.

## Step 1: Create Orders via API

When a sales order is created in your ERP, you can use the FleetOps API to automatically create a corresponding delivery order in Fleetbase.

You will need a **Secret Key** from the [Developer Console](/docs/platform/developer-console/api-keys).

```bash
curl -X POST https://api.fleetbase.io/v1/fleet-ops/orders \
  -H "Authorization: Bearer flb_test_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "internal_id": "ERP-SO-12345",
    "payload": {
      "pickup": {
        "name": "Main Warehouse",
        "location": {
          "type": "Point",
          "coordinates": [-122.395, 37.793]
        }
      },
      "dropoff": {
        "name": "Customer Address",
        "location": {
          "type": "Point",
          "coordinates": [-122.408, 37.783]
        }
      }
    }
  }'
```

Notice the `internal_id` field. You can use this field to store your ERP's sales order ID, making it easy to cross-reference records between the two systems.

## Step 2: Sync Statuses via Webhooks

To update your ERP when the delivery status changes in Fleetbase (e.g., when the driver marks the order as "Completed"), you should set up a [Webhook](/docs/platform/developer-console/webhooks).

1. Go to the Developer Console in Fleetbase.
2. Create a new Webhook.
3. Set the URL to an endpoint on your ERP or middleware server (e.g., `https://api.your-erp.com/webhooks/fleetbase`).
4. Select the events you want to listen for, such as `order.completed`, `order.en_route`, or `order.canceled`.

When the event occurs, Fleetbase will send a POST request to your URL with the order details:

```json
{
  "event": "order.completed",
  "data": {
    "id": "order_xxxxxxxx",
    "internal_id": "ERP-SO-12345",
    "status": "completed",
    "driver": "driver_xxxxxxxx",
    "completed_at": "2023-10-27T14:30:00Z"
  }
}
```

Your server can then parse this payload, extract the `internal_id`, and update the corresponding sales order in your ERP.

## Step 3: Sync Inventory (Optional)

If you are using the [Pallet](/docs/pallet/overview) extension for warehouse management, you can also sync inventory levels between Fleetbase and your ERP.

When inventory is received or shipped in Fleetbase, you can use webhooks (e.g., `inventory.updated`) to notify your ERP.

Conversely, when inventory is adjusted in your ERP, you can use the Pallet API to update the stock levels in Fleetbase.
