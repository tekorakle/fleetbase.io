# Build a Custom Extension

This guide walks you through the process of building a custom Fleetbase extension from scratch.

We will build a simple "Vehicle Inspection" extension that allows drivers to submit daily inspection reports, and fleet managers to view them in the console.

## Prerequisites

- A local Fleetbase development environment set up.
- The [Fleetbase CLI](/docs/extending-fleetbase/scaffolding-your-extension) installed (`npm install -g @fleetbase/cli`).
- Basic knowledge of PHP (Laravel) and JavaScript (Ember.js).

## Step 1: Scaffold the Extension

Use the Fleetbase CLI to generate the boilerplate code for your extension.

```bash
flb create:extension vehicle-inspection
```

This will create a new directory `vehicle-inspection` containing both the backend (Laravel package) and frontend (Ember engine) code.

## Step 2: Define the Backend Model

Navigate to the backend directory (`vehicle-inspection/server`) and create a new model and migration for the inspection reports.

```bash
php artisan make:model InspectionReport -m
```

In the generated migration file, define the schema:

```php
Schema::create('inspection_reports', function (Blueprint $table) {
    $table->uuid('id')->primary();
    $table->uuid('vehicle_uuid');
    $table->uuid('driver_uuid');
    $table->boolean('passed');
    $table->text('notes')->nullable();
    $table->timestamps();
});
```

Run the migration:

```bash
php artisan migrate
```

## Step 3: Create the API Controller

Create a controller to handle API requests for the inspection reports.

```bash
php artisan make:controller InspectionReportController
```

Add methods to list, create, and view reports. Ensure you register these routes in your extension's `routes/api.php` file.

## Step 4: Build the Frontend UI

Navigate to the frontend directory (`vehicle-inspection/addon`) and create a new route to display the reports.

```bash
ember generate route reports
```

In the generated template (`addon/templates/reports.hbs`), build a table to display the inspection reports fetched from your API.

## Step 5: Register the Menu Item

To make your extension accessible from the Fleetbase console sidebar, you need to register a menu item using the [Universe Service](/docs/extending-fleetbase/menus-and-navigation).

In your extension's `addon/engine.js` file, inject the `universe` service and register the menu item:

```javascript
export default class VehicleInspectionEngine extends Engine {
  // ...
  
  setupExtension() {
    this.universe.registerMenuItem('operations', {
      title: 'Inspections',
      icon: 'clipboard-check',
      route: 'vehicle-inspection.reports'
    });
  }
}
```

## Step 6: Test Locally

Link your extension to your local Fleetbase instance using the CLI:

```bash
flb link vehicle-inspection
```

Start your local Fleetbase server and Ember development server. You should now see the "Inspections" menu item in the sidebar, and clicking it should load your new route.

## Step 7: Publish to the Marketplace

Once you are satisfied with your extension, you can publish it to the Fleetbase Registry.

```bash
flb publish
```

Follow the prompts to authenticate and upload your extension. Once published, other Fleetbase users can install it via the console or CLI.
