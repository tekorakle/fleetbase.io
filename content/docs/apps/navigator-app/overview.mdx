# Navigator App Overview

The **Navigator App** is an open-source, React Native (Expo) mobile application designed for drivers and delivery personnel. It integrates directly with the [FleetOps](/docs/fleetops/overview) extension to provide a complete end-to-end delivery management solution.

## Key Features

- **Order Management**: Drivers can view incoming, active, and completed orders.
- **Turn-by-Turn Navigation**: Integrates with Google Maps, Apple Maps, and Waze for seamless routing to pickup and dropoff locations.
- **Proof of Delivery (POD)**: Collect electronic signatures, take photos, or scan QR codes to verify delivery completion.
- **Real-Time Tracking**: Broadcasts the driver's GPS location back to the Fleetbase console via SocketCluster, allowing dispatchers and customers to track the delivery in real-time.
- **Offline Support**: Caches order data locally so drivers can continue working even in areas with poor cellular connectivity.
- **Vehicle Management**: Drivers can select which vehicle they are currently operating.
- **Push Notifications**: Receive instant alerts when new orders are dispatched or when order details change.

## Supported Platforms

Because the Navigator App is built with React Native and Expo, a single codebase compiles to native applications for both major mobile platforms:

- **iOS**: Requires iOS 13.0 or later.
- **Android**: Requires Android 6.0 (API level 23) or later.

## Integration with FleetOps

The Navigator App acts as the "client" for the FleetOps "server".

When a dispatcher assigns an order to a driver in the Fleetbase console, the FleetOps API sends a push notification to the driver's device. The driver can then accept the order in the Navigator App, which updates the order status in FleetOps to `driver_assigned`.

As the driver progresses through the delivery (e.g., "En Route to Pickup", "Picked Up", "En Route to Dropoff", "Completed"), the Navigator App sends API requests to FleetOps to update the order status and trigger any associated webhooks.
