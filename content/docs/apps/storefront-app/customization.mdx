# Customization & White-Labeling

Once you have the Storefront App running locally, you can customize it to match your company's branding before deploying it to the App Store or Google Play Store.

## App Configuration (`app.json`)

The `app.json` file in the root directory contains the core configuration for Expo. This is where you define the app's name, bundle identifier, and visual assets.

```json
{
  "expo": {
    "name": "My Storefront App",
    "slug": "my-storefront-app",
    "version": "1.0.0",
    "orientation": "portrait",
    "icon": "./assets/icon.png",
    "splash": {
      "image": "./assets/splash.png",
      "resizeMode": "contain",
      "backgroundColor": "#ffffff"
    },
    "ios": {
      "bundleIdentifier": "com.mycompany.storefrontapp",
      "buildNumber": "1.0.0"
    },
    "android": {
      "package": "com.mycompany.storefrontapp",
      "versionCode": 1
    }
  }
}
```

### Key Fields to Update:
- `name`: The name of the app as it will appear on the user's home screen.
- `slug`: The URL slug for your Expo project.
- `ios.bundleIdentifier`: The unique identifier for your iOS app (e.g., `com.yourcompany.appname`).
- `android.package`: The unique identifier for your Android app.

## Visual Assets

Replace the default images in the `assets` folder with your own branding:

- `icon.png`: The app icon (1024x1024px).
- `splash.png`: The splash screen image displayed while the app loads.
- `adaptive-icon.png`: The Android adaptive icon.

After replacing these files, Expo will automatically generate the correctly sized icons for all devices during the build process.

## Theme and Colors

To change the primary colors used throughout the app (buttons, headers, links), locate the theme configuration file.

In the Storefront App, the theme is typically defined using Tailwind CSS classes or a central theme object in `src/utils/theme.js`.

```javascript
export const theme = {
  colors: {
    primary: '#3B82F6', // Change this to your brand's primary color
    secondary: '#10B981',
    background: '#F3F4F6',
    text: '#1F2937',
  }
};
```

Update these hex codes to match your brand guidelines.

## Building for Production

Once your customizations are complete, you can build the standalone binaries for iOS and Android using Expo Application Services (EAS).

1. Install the EAS CLI:
   ```bash
   npm install -g eas-cli
   ```

2. Log in to your Expo account:
   ```bash
   eas login
   ```

3. Configure the project for EAS:
   ```bash
   eas build:configure
   ```

4. Build the app for iOS and Android:
   ```bash
   eas build --platform all
   ```

EAS will handle the complex build process in the cloud. Once finished, it will provide links to download the `.ipa` (iOS) and `.aab` (Android) files, which you can then submit to the respective app stores.
