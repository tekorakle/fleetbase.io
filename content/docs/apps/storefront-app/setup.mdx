# Setup & Configuration

This guide explains how to set up the Storefront App locally, configure it to connect to your Fleetbase instance, and prepare it for deployment.

## Prerequisites

Before you begin, ensure you have the following installed on your development machine:
- **Node.js** (v16 or later)
- **Yarn** or **npm**
- **Expo CLI** (`npm install -g expo-cli`)
- **Git**

You will also need a **Public Key** from the [Developer Console](/docs/platform/developer-console/api-keys) in your Fleetbase instance.

## Step 1: Clone the Repository

Clone the open-source Storefront App repository from GitHub:

```bash
git clone https://github.com/fleetbase/storefront-app.git
cd storefront-app
```

## Step 2: Install Dependencies

Install the required Node.js packages using Yarn:

```bash
yarn install
```

## Step 3: Configure Environment Variables

Create a `.env` file in the root directory of the project. You can copy the provided `.env.example` file if it exists.

```bash
cp .env.example .env
```

Open the `.env` file and configure the following variables:

```bash
# The base URL of your Fleetbase API
API_URL=https://api.fleetbase.io/v1

# Your Fleetbase Public Key
PUBLIC_KEY=flb_test_xxxxxxxxxxxx

# The URL of your Fleetbase SocketCluster server (for real-time tracking)
SOCKET_URL=wss://socket.fleetbase.io/socketcluster/

# Optional: Stripe Publishable Key (for credit card payments)
STRIPE_KEY=pk_test_xxxxxxxxxxxx

# Optional: Google Maps API Key (for store locators and address autocomplete)
GOOGLE_MAPS_API_KEY=AIzaSyxxxxxxxxxxxx
```

*(If you are self-hosting Fleetbase, replace the `API_URL` and `SOCKET_URL` with your own domain).*

## Step 4: Run the App Locally

Start the Expo development server:

```bash
yarn start
```

This will open the Expo Developer Tools in your browser. From here, you can:
- Run the app in an iOS Simulator (requires Xcode).
- Run the app in an Android Emulator (requires Android Studio).
- Scan the QR code with the **Expo Go** app on your physical iOS or Android device to test it live.

## Step 5: Browse Your Store

To test the app, you need at least one active store with products in your Fleetbase instance.

1. Go to the Fleetbase Console and navigate to **Storefront > Stores**.
2. Create a new store and ensure it is marked as "Active".
3. Navigate to **Storefront > Products** and add a few products to your store.
4. Open the Storefront App on your emulator or device.

You should now see your store and products listed in the app. You can add items to your cart and proceed to checkout to test the full flow.

## Next Steps

Now that the app is running locally, you can proceed to [Customization](/docs/apps/storefront-app/customization) to white-label the app with your own branding before deploying it to the App Store or Google Play Store.
