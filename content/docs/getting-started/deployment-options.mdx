# Deployment Options

Fleetbase is designed to be highly flexible, offering multiple deployment options to suit your organization's infrastructure requirements, technical expertise, and data sovereignty needs.

Whether you want a fully managed solution or complete control over your servers, Fleetbase supports the following deployment models:

## 1. Fleetbase Cloud (Fully Managed)

**Best for:** Quick starts, startups, and organizations without dedicated DevOps teams.

Fleetbase Cloud is the official, fully managed SaaS offering. It provides instant access to the Fleetbase operating system without the need to provision servers, manage databases, or handle updates.

- **Setup Time:** Instant
- **Maintenance:** Fully managed by the Fleetbase team
- **Scaling:** Automatic
- **Extensions:** Install directly from the marketplace UI
- **Getting Started:** See the [Quickstart (Cloud)](/docs/getting-started/quickstart-cloud) guide.

## 2. Docker (Local Development)

**Best for:** Developers, testing, and evaluating the platform.

For developers building custom extensions or evaluating Fleetbase locally, the Docker-based installation provides a complete, isolated environment on your machine.

- **Setup Time:** ~5 minutes
- **Maintenance:** Self-managed
- **Requirements:** Docker, Docker Compose, Node.js
- **Getting Started:** Run `flb install-fleetbase` using the [Fleetbase CLI](/docs/getting-started/quickstart-self-hosted).

## 3. Cloud Self-Hosted (AWS, GCP, Azure)

**Best for:** Production environments requiring data sovereignty and custom infrastructure.

You can deploy Fleetbase to your own cloud infrastructure (AWS, Google Cloud, Azure, DigitalOcean, etc.). This gives you full control over your data, security policies, and network configuration while leveraging scalable cloud services.

- **Setup Time:** 30–60 minutes
- **Maintenance:** Self-managed (requires DevOps expertise)
- **Architecture:** Typically deployed using Kubernetes (EKS, GKE, AKS) or Docker Swarm, with managed database services (RDS, Cloud SQL) and managed Redis (ElastiCache, Memorystore).
- **Getting Started:** See the [Quickstart (Self-Hosted)](/docs/getting-started/quickstart-self-hosted) guide for the foundational Docker setup, which can be adapted for cloud orchestration.

## 4. On-Premise (Bare Metal)

**Best for:** Enterprise organizations with strict compliance, security, or air-gapped requirements.

Fleetbase can be installed directly on bare-metal servers or private data centers. This option provides the highest level of control and isolation.

- **Setup Time:** 30–60 minutes
- **Maintenance:** Self-managed (requires dedicated IT/infrastructure teams)
- **Requirements:** Linux servers (Ubuntu/Debian recommended), MySQL 8+, Redis, PHP 8+, Node.js.
- **Getting Started:** Use the [Fleetbase CLI](/docs/getting-started/quickstart-self-hosted) or the provided installation scripts in the main repository.

## Choosing the Right Option

| Feature | Fleetbase Cloud | Cloud Self-Hosted | On-Premise |
|---|---|---|---|
| **Data Control** | Managed | Full Control | Full Control |
| **Infrastructure Management** | None | High | Very High |
| **Custom Extensions** | Yes (via Registry) | Yes | Yes |
| **Direct Database Access** | No | Yes | Yes |
| **Compliance (HIPAA, SOC2)** | Shared Responsibility | Customer Managed | Customer Managed |
| **Cost Structure** | Subscription | Infrastructure Costs | Hardware Costs |

If you are unsure which option is best for your organization, we recommend starting with **Fleetbase Cloud** to evaluate the platform, as you can always migrate to a self-hosted environment later.
